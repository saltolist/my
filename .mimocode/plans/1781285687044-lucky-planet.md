# Исправление: тряска строк при наведении во время перетаскивания элемента в заметке

## Текущее состояние кода

Фикс **уже применён**. Старый класс `.note-body-doc-drop-break` (in-flow `display: block` с `margin: 12px 0`) больше нигде не используется.

Текущая реализация в `NoteBodyDocumentView.tsx` (строки 31-44):
```tsx
function renderLineLeadSlot(...) {
  return (
    <div className="note-embed-drop-lead-abs" aria-hidden>
      <NoteBodyDropIndicator axis="horizontal" />
    </div>
  );
}
```

- Lead slot рендерится **внутри** контейнера строки (строка 136 для текстовых, строка 123 для embed)
- `.note-body-doc-line--text` имеет `position: relative` (shell-notes.css:499)
- `.note-embed-drop-lead-abs` позиционируется абсолютно: `position: absolute; top: 0; height: 0; pointer-events: none`
- Дочерний индикатор: `position: absolute; margin: 0 !important; transform: translateY(-50%)`

Это идентична паттерну из legacy `NoteBodyCanvas`.

## Если баг всё ещё воспроизводится

Если тряска осталась, проверить:

1. **`dropUtils.ts:155-217` — `hitTestLine()`**: На каждое `pointermove` выполняется `querySelectorAll` + `getBoundingClientRect()` на всех `[data-line]` элементах. Синхронный layout thrashing может вызывать визуальные артефакты. Решение — кэшировать rects или throttle.

2. **`useNoteBodyEmbedDrop.ts:217-241`**: Каждое движение мыши вызывает `setDropBefore()`, что перерисовывает весь `NoteBodyDocumentView`. Если state обновляется слишком часто, React может не успевать батчить обновления.

3. **`dropUtils.ts:94-132` — `stabilizeDropBeforeCommitted()`**: Гистерезис `textLineDropHysteresisPx()` = `Math.max(28, lineHeight * 0.28)`. Если lineHeight маленький, гистерезис может быть недостаточным для предотвращения осцилляций на границе строк.

4. **CSS transitions**: Убедиться, что во время драга отключены transition на текстовых строках (сейчас `transition: none` применяется только к image-grid ячейкам — `shell-notes.css:332-335`).
