## Резюме

Проект — Next.js 16 SPA (статический экспорт для GitHub Pages) на React 19, построенный по **Feature-Sliced Design** с Zustand, TanStack Query, Tailwind CSS v4 и shadcn/ui. Кодовая база **качественная и хорошо структурированная**, но имеет несколько серьёзных проблем, требующих внимания.

## Критические проблемы (5)

### 1. XSS-уязвимость: `dangerouslySetInnerHTML` без санитизации
- `src/widgets/chat-thread/ui/ChatAiMessage.tsx:33`
- `src/widgets/chat-thread/ui/ChatUserMessage.tsx:109`
- HTML от AI и пользователей рендерится без DOMPurify

### 2. `usePost(id)` загружает ВСЕ посты
- `src/entities/post/model/usePosts.ts:17-30`
- Нет серверного метода `get(id)`, каждый post detail запрашивает весь список
- **Масштабируемый узкое место** — при 1000+ постах будет неработоспособно

### 3. `copyPlainText` дублируется 3 раза
- `src/entities/message/lib/utils.ts:41` (каноническая)
- `src/widgets/post-workspace/ui/PostCardToolbar.tsx:7`
- `src/widgets/chat-thread/ui/AiMessageToolbar.tsx:84`

### 4. `CloseIcon` компонент дублируется 3 раза
- `src/widgets/profile-settings/ui/telegram/TelegramInputs.tsx:58`
- `src/widgets/profile-settings/ui/user/UserEmailCodeInput.tsx:3`
- `src/widgets/page-header/ui/PageHeaderSearchInput.tsx:26`

### 5. 55 объявлений `!important` в CSS
- Худший нарушитель: `shell-notes.css` — 20 штук (строки 438-460)
- Индикатор specificity wars

## Архитектурные находки

### FSD compliance: Excellent
- Все 6 слоёв соблюдены
- `eslint-plugin-boundaries` автоматически enforced
- Единственное нарушение: циклическая зависимость `widgets/feed` ↔ `widgets/post-workspace`

### Циклическая зависимость виджетов
- `widgets/feed/index.ts` экспортирует `PostCommentsRow` из `widgets/post-workspace`
- `widgets/post-workspace` импортирует из `widgets/feed`
- Решение: вынести общие компоненты (`PostReactionPills`, `PostViewsReposts`, `PostCommentsRow`) в `entities/post` или `shared/ui`

### Feature-to-feature импорт
- `features/post-context-menu` импортирует из `features/schedule-post`
- FSD best practice запрещает coupling между features

### Widget sidebar импортирует из features
- `widgets/sidebar/model/useSidebar.ts:22` импортирует `usePostCtxMenuItems` из `features/post-context-menu`
- Архитектурно: features должны потребляться screens, не widgets

### ProfileHydrator в App Providers
- `app/providers/AppProviders.tsx:29` рендерит `ProfileHydrator` из `widgets/profile-settings`
- Нарушение: app не должен содержать виджеты

## Масштабируемость: узкие места при 10x

| Проблема | Влияние | Решение |
|---|---|---|
| `usePost` грузит все посты | Критическое | Добавить `get(id)` в `PostsRepository` |
| Нет code splitting | Критическое | `React.lazy()` для analytics, charts, note-editor, profile |
| God-hook `useSidebar` (343 строки) | Высокое | Декомпозиция на подхуки |
| `NavigationState` как god object | Высокое | Вынести screen-specific фильтры в локальный стейт |
| `shared/lib` как ящик для всего | Среднее | Перенести domain-код в entities |
| Нет пагинации/виртуализации | Среднее | `@tanstack/react-virtual` или infinite scroll |
| Нет `React.memo` | Среднее | Мемоизировать list-item компоненты |
