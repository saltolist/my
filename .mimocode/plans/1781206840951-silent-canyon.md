## 2. Нарушения разделения ответственности (SRP)

### 2.1 [HIGH] `composer-store.tsx` — смешение 5+ ответственности

Файл в `app/model/store/` но это **React Context Provider** (JSX), не Zustand store. Смешивает:
- Навигационный мост (registerNavBridge)
- Валидацию LLM (assertLlm + window.alert)
- Создание чатов с async мутациями
- Мок AI-ответов через `window.setTimeout`
- Навигацию (goToHref)

**Действие**:
1. Переместить из `app/model/store/` в `app/providers/composer-provider.tsx` (это Provider, не store)
2. Вынести бизнес-логику отправки (`sendHome`, `sendGChat`, `sendPost`) в `shared/lib/composerService.ts`
3. Заменить `window.alert` на возврат ошибки, которую UI обработает

### 2.2 [HIGH] `navigation-store.ts` — "god store"

`NavigationState` содержит 12 полей из 5 разных доменов:
- Route: `currentPostId`, `isEditing`
- Note: `currentNote`, `noteMode`, `noteFrom`, `noteSavedSnapshot`, `noteScope`, `noteFilter`, `notesSearch`
- Chat: `chatsTab`, `chatsSearch`
- Feed: `feedSearch`
- Analytics: `analyticsPeriod`

Любой подписчик на любое поле ре-рендерится при изменении любого другого.

**Действие**: Разбить на per-screen stores (`noteNavigationStore`, `feedNavigationStore`, `chatsNavigationStore`), оставить в `navigation-store` только route-состояние (`currentPostId`, `isEditing`).

### 2.3 [MEDIUM] `useNoteEditor` (225 строк) — mixed concerns

Хук смешивает:
- UI-состояние (title, body, files, bodyFocusRequest)
- Бизнес-логику сохранения (save с 4 ветками)
- Навигацию (setViewMode, setEditMode)
- Side effects (registerNotePersist)

Функция `save` (строки 100-171) — 70 строк с 4 ветками и 12-элементным deps массивом.

**Действие**: Вынести `save` в отдельный `useNoteSave` хук.

### 2.4 [MEDIUM] `usePostWorkspace` (239 строк) — god hook

15 внешних хуков, 25+ возвращаемых полей. Управляет навигацией, CRUD поста, заметками, чатами, edit-mode, контекстным меню, layout'ом, скроллом.

**Действие**: Разбить на:
- `usePostNavigation` (route/mode sync)
- `usePostActions` (CRUD, edit, notes)
- `usePostLayout` (refs, scroll, responsive)

### 2.5 [MEDIUM] `useNoteScreen` и `useNoteEditor` — пересечение ответственности

Оба хука управляют заметками, оба вызывают `setNav`, оба имеют `patchNote` обёртку. AI toggle реализован дважды: `setNoteAi` в `useNoteScreen` (строки 90-107) и `toggleNoteAi` в `usePostWorkspace`.

**Действие**: Выделить общие note-операции в `entities/note/model/useNoteOperations.ts`.

## 4. Отсутствие обработки ошибок

### 4.1 [HIGH] Entity mutation hooks без error handling

Все `useCallback`-обёртки в `src/entities/post/model/` не имеют `try/catch`:
- `useLocalChatMutations.ts` — 2 хука
- `usePostNoteMutations.ts` — 4 хука
- `usePostCommentMutations.ts` — 1 хук

Если `updatePost.mutateAsync()` бросит ошибку — `setCachedPost()` не выполнится, кэш останется в inconsistent-состоянии.

**Действие**: Обернуть в `try/catch` с rollback кэша и уведомлением пользователя.

### 4.2 [MEDIUM] Fire-and-forget в `composer-store.tsx`

Строки 174, 179, 183-186 — `void addLocalChat(...).then(...)` без `.catch()`. Если `addLocalChat` упадёт, `setTimeout` всё равно попытается пушить сообщение в несуществующий чат.

### 4.3 [MEDIUM] `httpClient.ts` — unsafe casts

Строки 50, 53: `return undefined as T` — небезопасно. Тип возврата `Promise<T>` не отражает реальность.

**Действие**: Изменить на `Promise<T | undefined>`.

## 5. Непоследовательные паттерны

### 5.1 [MEDIUM] Два разных API для мутаций в entities

- `useCreatePost` возвращает **mutation object** (`useMutation` directly)
- `useAddPostNote` возвращает **callback function** (обёртка через `useCallback`)

Вызывающий код не может единообразно компонировать мутации: `addPostNote(id, note)` vs `createPost.mutate(newPost)`.

**Действие**: Выбрать один паттерн. Рекомендация: везде использовать `useMutation` (стандарт React Query).

### 5.2 [MEDIUM] Смешанные соглашения именования файлов

В `widgets/sidebar/ui/` и `widgets/page-header/ui/`:
- PascalCase: `Sidebar.tsx`, `PageHeader.tsx`
- kebab-case: `sidebar-header.tsx`, `page-header-menu-button.tsx`

**Действие**: Выбрать одно соглашение. В FSD принят kebab-case для UI-слайсов.

### 5.3 [LOW] Непоследовательная навигация

- `useSidebar.ts` оборачивает `router.push` в `goTo` callback
- `usePostWorkspace.tsx`, `useFeedScreen.tsx`, `useNoteScreen.tsx` используют сырой `router.push()`

**Действие**: Создать `useAppNavigation` хук с единым API.

### 5.4 [LOW] No-op стабы в `useChatMessage.ts`

Строки 55-75: 11 функций-заглушек (`setDraft: () => undefined`, `startEdit: () => undefined` и т.д.). Компонент получает пропсы, которые ничего не делают.

**Действие**: либо реализовать, либо убрать из интерфейса.

## 6. Проблемы с архитектурой

### 6.1 [HIGH] Мок AI-логики в продакшн-коде

`composer-store.tsx` импортирует `getGlobalReply`, `getPostReply`, `buildAiReplyMessage` и использует их с `window.setTimeout`. Это мок-логика, смешанная с реальными мутациями.

**Действие**: Вынести в `src/shared/lib/aiSimulation.ts`, инжектировать через условие `if (API_MODE === 'seed')`.

### 6.2 [MEDIUM] MSW в production dependencies

`msw` в `dependencies` (не `devDependencies`). Увеличивает bundle size.

**Действие**: Рассмотреть dynamic import с `ssr: false` или tree-shaking.

### 6.3 [MEDIUM] Zod-схемы не подключены к валидации

Схемы в `shared/api/schemas/` определены, но `apiRequest` делает `JSON.parse(text) as T` без валидации.

**Действие**: Добавить опциональную Zod-валидацию в `apiRequest`.

### 6.4 [LOW] `Date.now()` для генерации ID

`composer-store.tsx:114,165` и `useNoteEditor.tsx:108,123` используют `Date.now()` для ID. Уязвимо к коллизиям при rapid actions.
