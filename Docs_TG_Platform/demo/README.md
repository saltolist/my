# TG Platform demo

Демо разбито на простые статические файлы без сборщика. `index.html` можно обслуживать любым статическим сервером, а логика подключается обычными `<script>`.

## Структура

- `index.html` — HTML-оболочка: sidebar, экраны и контейнеры для динамической отрисовки.
- `styles/main.css` — все стили интерфейса.
- `scripts/data.js` — моковые посты, чаты, заметки и настройки ИИ.
- `scripts/state.js` — текущие выбранные сущности и UI-состояния.
- `scripts/navigation.js` — переключение экранов.
- `scripts/composer.js` — выбор LLM/Web Search моделей, мультиответ и настройки ИИ.
- `scripts/feed.js` — лента и создание черновика.
- `scripts/post.js` — пространство поста, режимы чат/заметки, публикация и вложения.
- `scripts/notes-page.js` — страница отдельной заметки.
- `scripts/chats.js` — список глобальных и локальных чатов.
- `scripts/notes-list.js` — каталог заметок.
- `scripts/ai-chat.js` — глобальный чат и моковые ответы ИИ.
- `scripts/post-chat.js` — локальный чат поста и моковые ответы ИИ.
- `scripts/menus.js` — контекстные меню и удаления.
- `scripts/profile.js` — вкладки профиля.
- `scripts/helpers.js` — общие утилиты.
- `scripts/init.js` — стартовая инициализация.

## Как развивать полный сценарий

Для полноценной демо-версии по `concept/09-user-scenario.md` удобнее двигаться актами:

1. Первый запуск: пустое состояние, подключение Telegram, настройка OpenAI, заполнение профиля.
2. Утро: создание заметки, обсуждение заметки с ИИ, включение заметки в контекст.
3. День: создание черновика из заметки, итерации с локальным ИИ, сохранение.
4. Публикация: ручная публикация, метрики поста, объяснение результата ИИ.
5. Конец недели: глобальный анализ, контент-план, создание черновика из пункта плана.
6. Технический контроль: история запросов ИИ, стоимость, уточнение правил профиля.

На следующем шаге лучше вынести сценарные мок-данные и ответы ИИ в отдельный слой, например `scripts/scenario.js`, чтобы экраны могли показывать разные состояния одного пользовательского пути.
