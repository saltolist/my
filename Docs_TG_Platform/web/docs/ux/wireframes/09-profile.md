# Профиль (`profile`)

**Reference:** `web-legacy/src/screens/profile/`, `web-legacy/src/widgets/profile-settings/`

Экран разделён на три вкладки: **`Настройки` | `Канал` | `Аналитика платформы`**.

**Стартовая вкладка: `Настройки`** (index 0).

```
┌─────────────────────────────────────────────────────────────────┐
│  Профиль канала    [Настр.|Канал|Аналит.]           [← Назад]   │
│ ─────────────────────────────────────────────────────────────── │
│  (содержимое активной вкладки)                                  │
└─────────────────────────────────────────────────────────────────┘
```

## Шапка

- Заголовок `Профиль канала`.
- Вкладки — в центральном слоте `PageHeader`.
- Справа: `← Назад` → `home`.
- На вкладке «Аналитика платформы» при узкой шапке — picker периода в header.

## Несохранённые изменения

`window.confirm` при уходе с вкладки **`Настройки`** или **`Канал`** с dirty-state.

## Вкладка `Канал`

Редактируемая база знаний (см. [profile-settings.md](../components/widgets/profile-settings.md#channeltab)):

### Ядро канала

- О чём канал, Цель канала
- Целевая аудитория, Ценность для аудитории
- Портрет автора

### Голос и формат

- Тон, Обращение к читателю
- Базовый формат поста

### Правила

- Обязательно, Избегать

### Рубрики

- Название + описание на каждую рубрику; добавление и удаление строк

Кнопки: `Сохранить` / `Отменить` (при dirty).

Концептуальный гайд по содержанию полей — [05-channel-profile.md](../../product/05-channel-profile.md).

## Вкладка `Настройки`

### Тема

Секция **«Тема»**: `☀ Светлая`, `🖥 Системная`, `🌙 Тёмная`. Сохранение в `localStorage`, `data-theme` на `<html>`.

### ИИ-движок

Группы моделей:

| Группа | Описание |
|---|---|
| LLM-модели | Provider + model, API key, «Включить в мультиответ», активная модель |
| Web Search модели | Аналогично |
| Оркестратор | Exclusive model, без multi-toggle |
| Web Reasoner | Exclusive model |
| RAG Reasoner | Exclusive model |

- Общий переключатель **Мультиответ** (нужно ≥2 пар LLM+Web с multi-flag).
- **Системный промпт** — textarea.

Параметры `temperature` / `max tokens` в типах не используются — **поля не рендерятся**.

### Telegram

- `api_id`, `api_hash` (save/cancel)
- Телефон, код подтверждения, resend
- Подключение канала
- **`sessionName` и `syncMode`** — только в seed/types, **не рендерятся** в форме
- **Омниканальный бот** — отдельная секция (`TelegramOmnibotSection`) с bot API token
- `Сбросить настройки`
