# Пользовательский сценарий

> **Нarrative example.** Сценарий иллюстрирует типичный день автора и **не описывает каждый клик буквально**. Актуальные экраны, поля и кнопки — в [pages.md](../ux/pages.md) и product-доках §05–07.  
> В local-first фазе при первом открытии уже есть **seed-данные** (лента, профиль, заметки), а не пустой экран.

Полный сквозной сценарий: от настройки канала до публикации поста, анализа и планирования недели.

---

## Персонаж

**Антон** — автор канала «Деньги без паники» (~8 400 подписчиков). Личные финансы: инвестиции, бюджет, психология денег. Публикует 3–4 поста в неделю.

---

## Акт 1. Первый запуск — настройка платформы

Антон разворачивает платформу. После входа видит **лента с seed-постами** и заполненный черновик профиля — это демо-данные local-first фазы.

### 1.1 Подключение канала

`Профиль` → вкладка **`Настройки`** → блок **Telegram**: api_id, api_hash, авторизация по телефону, выбор канала. Отдельно — **Омниканальный бот** (Bot API token).

После реального подключения лента может подтянуть посты канала; в demo-режиме работа идёт на seed.

### 1.2 Настройка ИИ-движка

Там же, **`Настройки`**: добавляет LLM-модель, вставляет API-ключ, при необходимости включает **мультиответ** и правит **системный промпт**. Сохраняет.

### 1.3 Заполнение профиля канала

`Профиль` → вкладка **`Канал`**. Поля — textarea, без отдельных форм «демография» / «бренд-бук»:

**Ядро канала.** В «ЦА» текстом описывает читателя: 25–38 лет, крупные города, найм, страх ошибиться с инвестициями, не любит инфобизнес.

**Рубрики.** Четыре строки, у каждой **название + описание**:
- «Разбор» — инструмент или явление
- «Психология денег» — поведение вокруг финансов
- «Личный опыт» — что сделал сам
- «Новости» — комментарий к повестке

Частоту и примеры постов в UI не задаёт — при необходимости пишет это в **описании** рубрики или в «Базовый формат поста».

**Голос и правила.** В «Тон» и «Обращение» — разговорный стиль, от первого лица. В «Избегать» — «профит», «иксы», крипта, политика, конкретные брокеры.

Сохраняет. Дальнейшие ответы ИИ опираются на эти поля.

---

## Акт 2. Утро — идея для поста

Антон открывает **`Ленту`**, смотрит охваты вчерашнего поста про ИИС в карточке (👁, реакции).

### 2.1 Заметка из головы

`Заметки` → **`+ Новая заметка`**. Фиксирует мысль про страх открыть брокерский счёт. Сохраняет **без** AI-флага — сырая идея.

### 2.2 Развитие идеи с ИИ

Переходит на **`home`** (глобальный чат) или открывает сохранённый **`gchat`**. В composer можно прикрепить пост через **`@`** (синтаксис `@<title>;`).

**Антон:** «Есть идея в заметках про страх открыть счёт. Это «Психология денег» или «Разбор»?»

**ИИ** опирается на профиль (рубрики, ЦА) и отвечает, что ближе «Психология денег», предлагает углы.

Антон возвращается в заметку → меню **`•••`** → **`Учитывать в ИИ`**, чтобы мысль попала в контекст канала.

---

## Акт 3. День — написание поста

### 3.1 Создание черновика

**`Лента`**: в composer внизу набирает текст или медиа → **`↑`** — черновик появляется в секции **Черновики**. Клик по карточке → пространство поста, режим **`chat`**.

### 3.2 Итерации в локальном чате

Карточка поста сверху, composer внизу. Attach: медиа поста, файл, медиа из прикреплённых постов.

**Антон:** просит хук, правит текст через «✏ Редактировать» на карточке или просит ИИ упростить формулировки в чате.

**ИИ** переписывает с учётом «Избегать» и тона из профиля.

### 3.3 Черновик без публикации

Пост остаётся **черновиком** в ленте. Публикация — позже через меню **`•••`** в шапке поста (**`Опубликовать`** / **`Запланировать`**).

---

## Акт 4. Вечер вторника — публикация и анализ

Открывает черновик → **`•••`** → **`Опубликовать`**.

### 4.1 Метрики поста

В ленте на карточке — просмотры и реакции. В пространстве поста — те же метрики в шапке/карточке. Спрашивает локальный ИИ, почему пост сработал.

### 4.2 Заметка из обратной связи

`Заметки` → новая глобальная заметка про «барьер первой сделки» — материал для серии.

---

## Акт 5. Конец недели — стратегия

**`home`** или **`gchat`**: глобальный чат.

**Антон:** «Что сработало за неделю?» — ИИ смотрит посты и метрики (seed/analytics).

**Антон:** «План на следующую неделю с учётом серии про барьеры» — ИИ предлагает черновики тем; при команде «создай черновик» может появиться новый пост в ленте (агентское действие).

---

## Акт 6. Технический контроль

`Профиль` → **`Аналитика платформы`**: график по моделям, таблица использования, активность (чаты, заметки, посты).

Уточняет профиль на вкладке **`Канал`** (например, в «Избегать» — «списки из 5 пунктов»).

**Нет в UI:** лайки/дизлайки ответов, история агентов, отдельный раздел «что ИИ понимает неправильно».

---

## Итог: полный цикл

| Действие | Где в UI |
|---|---|
| Telegram + ИИ-модели | `profile` → **Настройки** |
| База знаний канала | `profile` → **Канал** |
| Идея в заметку | `notes` / `note` |
| Диалог с глобальным ИИ | `home` / `gchat` |
| Черновик поста | `feed` → composer **`↑`** |
| Работа над постом | `post` → режим **`chat`** |
| Публикация | `post` → **`•••`** → **Опубликовать** |
| Аналитика канала | `analytics` |
| Метрики ИИ | `profile` → **Аналитика платформы** |
