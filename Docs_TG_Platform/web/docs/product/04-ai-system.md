# ИИ-система

Двухуровневая архитектура ИИ **надстраивается над [пространственной системой](./03-spaces.md)**: глобальный и локальный контуры соответствуют двум типам пространств.

> **UI reference:** `web-legacy`. Local-first фаза использует **stub-ответы** ИИ; таблицы «агентских действий» ниже описывают **целевое поведение по команде пользователя**, когда подключён реальный LLM.

---

## Локальный ИИ (уровень поста)

Каждое пространство поста содержит **локальные чаты** — отдельные переписки с помощником, привязанным к посту. Память между чатами одного поста **не делится**.

При входе в пространство активного чата нет (`currentPostChatId === null`). Первое сообщение **создаёт** чат. Список чатов — режим **`Чаты`**; **`+ Новый чат`** — пустой черновик.

**Контекст локального ИИ = локальная база + наследуемая глобальная:**

Своё:
- пост (текст, медиа, метрики, история правок)
- локальные заметки с **включённым** AI-флагом
- история **этого** локального чата

Наследуется:
- профиль канала (вкладка **`Канал`**: ядро, голос, правила, рубрики)
- системный промпт
- глобальные заметки с AI-флагом
- метрики канала

Чужие посты и чужие локальные заметки **не видны**.

**Типичные задачи:** доработать текст, хук, подпись к медиа, объяснить метрики поста, предложить формат.

**Агентские действия** (по явной команде, при реальном backend):

| Команда | Результат в UI |
|---|---|
| «Отредактируй пост…» | обновление текста в карточке поста |
| «Опубликуй этот пост» | статус → опубликован (меню **`•••`**) |

## Глобальный ИИ (уровень канала)

Глобальный чат на **`home`** / **`gchat`**. Видит контекст канала целиком.

**Контекст:**
- профиль канала
- глобальные заметки с AI-флагом
- метрики канала (`analytics`)
- посты — через **прикрепление в сообщении** (`@<title>;` → чип над composer), не отдельная кнопка «запинить»
- в attach-меню **`+`**: файл с компьютера, медиа из уже прикреплённых постов; **`pinnedPostIds`** в seed используются для подменю медиа из «избранных» постов, не для отдельного UI «закреплённых постов чата»

**Типичные задачи:** контент-план, сравнение постов, рекомендации по рубрикам из профиля, паттерны по метрикам.

**Агентские действия** (при реальном backend):

| Команда | Результат |
|---|---|
| «Создай черновик на тему X» | новый пост в секции **Черновики** ленты |
| «Опубликуй пост [название]» | смена статуса поста |

---

## Принцип работы агентов

Только **по команде пользователя**. Цикл: команда → действие → видимый результат в UI.

**Отдельного экрана истории агентов в reference client нет.** Контроль расходов — вкладка **`Аналитика платформы`** (графики моделей, таблица, активность), без like/dislike на ответы.

---

## Медиа в чате

| Scope | Attach через **`+`** |
|---|---|
| **`post` / локальный чат** | медиа текущего поста, файл, медиа из прикреплённых постов |
| **`home` / `gchat`** | файл, медиа из прикреплённых постов |

Посты в глобальный чат — через **`@` mention**, не через пункт attach-меню.

## Контекст ИИ (сводная таблица)

| Источник | Глобальный | Локальный |
|---|---|---|
| Профиль (`Канал`) | да | да (наследуется) |
| Системный промпт | да | да |
| Глобальные заметки с AI-флагом | да | да |
| Метрики канала | да | да |
| Пост в сообщении (`@`) | да | — |
| Сам пост + его метрики | через `@` | да |
| Локальные заметки с AI-флагом | — | да |
| История чата | свой чат | свой чат |

Локальные пространства друг друга не видят.

## Настройка ИИ-движка

`profile` → **`Настройки`** → блок **ИИ-движок**:

- LLM / Web Search / Оркестратор / Web Reasoner / RAG Reasoner
- API-ключи, **мультиответ**, **системный промпт**

Поля «температура» и «max tokens» **в форме не отображаются**.
