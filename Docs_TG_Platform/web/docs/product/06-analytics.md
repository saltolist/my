# Аналитика

В web-клиенте аналитика — **два экрана** с разным назначением. Ниже только то, что есть в UI reference (`web-legacy`).

Детали блоков: [pages.md § analytics](../ux/pages.md#8-analytics) · [wireframe](../ux/wireframes/08-analytics.md).

---

## Аналитика канала

**Маршрут:** `/analytics/` · пункт сайдбара **«Аналитика»**.

**Шапка:** `Аналитика канала`; `← Назад` → `home`.

**Период (5):** `24 часа`, `7 дней`, `30 дней`, `90 дней`, `Всё время`.  
На desktop picker — внутри карточки «Динамика прироста»; на mobile — в шапке.

### Блоки экрана

| # | Блок | Компонент | Содержимое |
|---|---|---|---|
| 1 | **Динамика прироста** | `ChannelAnalyticsSection` | Summary mini-metrics + `MultiSeriesTrendChart`; selector серий |
| 2 | **Прирост по метрикам** | `ChannelMetricBars` | Столбцы по метрикам за период |
| 3 | **Реакции** | `ChannelReactionsPanel` | Распределение реакций |
| 4 | **Тепловая карта активности** | `AnalyticsHeatmap` | Активность по времени |
| 5 | **Лучшие посты за период** | `AnalyticsTopPostsTable` | Таблица с переходом к посту (`→`) |

### Серии в графике

Через selector: просмотры, подписчики, реакции, комментарии, репосты, ER (`channelAnalyticsTrend.ts`).

Данные в local-first фазе — mock/seed (`analytics-seed.ts`).

**На этом экране нет:** виджета «По рубрикам», отдельного блока «рост из источников», UI like/dislike для ИИ.

## Аналитика платформы (ИИ)

**Маршрут:** `/profile/` → вкладка **«Аналитика платформы»** (не в основной навигации).

| Блок | Содержимое |
|---|---|
| `PlatformModelsChartSection` | График стоимости/использования по моделям; фильтр типа модели; период |
| `PlatformModelUsageSection` | Таблица суммарного использования по моделям |
| `PlatformActivitySection` | Мини-метрики активности: чаты, заметки, посты |

**На этой вкладке нет:** оценок ответов (лайк/дизлайк), истории агентских задач, лога запросов к ИИ построчно.

---

## Связь с ИИ

Метрики канала (экран `analytics`) и заполненность профиля — часть контекста глобального и локального ИИ. В чате можно спрашивать про тренды охватов, лучшие посты, время публикации — опираясь на те же seed/mock-данные, что видит экран.
