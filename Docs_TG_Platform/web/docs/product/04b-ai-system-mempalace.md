# ИИ-система — вариант с MemPalace

> **Не входит в legacy UI.** Документ описывает **целевую backend-архитектуру памяти** (MemPalace вместо классического RAG). Текущий reference client (`web-legacy`) — seed + stub ИИ без MemPalace.  
> Поведение ИИ в интерфейсе — [04-ai-system.md](./04-ai-system.md).

---

## Почему MemPalace, а не RAG с суммаризацией

Классический подход: посты сжимаются в краткие резюме и хранятся как векторы. При запросе достаются похожие чанки.

Проблема: суммаризацию делает ИИ, и он выбрасывает нюансы — авторский стиль, конкретные формулировки, контекст решений. Именно то, что нужно для точной работы с голосом канала.

**MemPalace хранит всё дословно.** Retrieval такой же — векторный поиск — но из индекса достаётся оригинальный текст, а не сжатая копия. Ничего не теряется при записи.

## Архитектура памяти

MemPalace использует трёхуровневую структуру, которая хорошо ложится на задачи платформы:

### Долгосрочная семантическая память
Хранит то, что почти не меняется:
- Профиль канала (поля вкладки **`Канал`**: ядро, голос, правила, рубрики)
- Заметки с флагом "включить в контекст"

Обновляется редко — только когда пользователь меняет профиль.

### Эпизодическая память
Каждый пост — отдельный эпизод. Хранится дословно со структурированными метаданными:
- Текст поста
- Рубрика, дата, статус (опубликован / черновик)
- Метрики (охваты, реакции, репосты)
- Теги

В терминах MemPalace: канал — это "крыло", рубрика — "комната", пост — "ящик".

### Рабочая память
Текущий контекст конкретного запроса:
- Для локального ИИ: полный текст текущего поста + несколько релевантных постов из эпизодической памяти
- Для глобального ИИ: профиль + релевантные эпизоды из истории канала

Retrieval собирает рабочую память под заданный токен-бюджет — пользователь контролирует, сколько памяти передаётся в модель.

## Как это меняет работу локального и глобального ИИ

### Локальный ИИ с MemPalace

Когда пользователь открывает пространство поста, платформа:
1. Берёт полный текст поста (рабочая память)
2. Запрашивает у MemPalace похожие посты из той же рубрики
3. Достаёт релевантные заметки
4. Собирает профиль канала

Модель получает оригинальные тексты похожих постов — видит реальный стиль автора, а не резюме о стиле.

### Глобальный ИИ с MemPalace

Для стратегических запросов ("проанализируй последний месяц", "составь контент-план"):
1. MemPalace достаёт релевантные эпизоды по смыслу запроса
2. Плюс временная фильтрация — например, только посты за последние 30 дней
3. Плюс фильтрация по метаданным — только посты рубрики "Психология денег"

Вместо передачи всей истории канала — точная выборка в рамках токен-бюджета.

## Место на диске vs токены в контексте

| | RAG с суммаризацией | MemPalace |
|---|---|---|
| Место на диске | Меньше — хранит резюме | Больше — хранит оригиналы |
| Токены на запрос | Зависит от retrieval | Зависит от retrieval |
| Качество retrieved контента | Теряет нюансы при записи | Оригинал без потерь |
| Стоимость записи | Тратит токены на суммаризацию | Бесплатно — просто сохраняет |

Место на диске — не проблема для self-hosted проекта. Токены в контексте в обоих случаях управляются одинаково — через токен-бюджет retrieval.

## Технический стек памяти

- **ChromaDB** — векторное хранилище для семантического поиска
- **SQLite** — метаданные постов (рубрика, дата, метрики, теги)
- **MemPalace retrieval** — комбинирует векторное сходство, временную близость и граф связей
- **Context Synthesizer** — собирает retrieved память в окно контекста под заданный бюджет токенов

Всё локально. Никаких внешних API для работы с памятью.

## Интеграция

MemPalace поддерживает MCP (Model Context Protocol), что позволяет подключить его к любому ИИ-движку без кастомной интеграции. Совместим с OpenAI, Anthropic, локальными моделями через Ollama.
