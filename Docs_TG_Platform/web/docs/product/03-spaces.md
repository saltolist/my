# Пространственная модель (web-клиент)

Документ фиксирует, как пространственная логика реализована в web-клиенте (`Docs_TG_Platform/web/`, Next.js + TS).

---

## Базовая идея

Платформа использует пространственную модель: вместо одного универсального чата — несколько изолированных пространств. У каждого пространства свои границы контекста, своя база знаний и свой ассистент.

Пространств два типа:

- **Глобальное** — одно на канал. Знает всё: свою базу знаний + все локальные пространства.
- **Локальное** — одно на пост. Знает свою базу знаний + **наследует всю не-сущностную часть глобальной**.

| Пространство | Привязка | Чаты | Заметки | База знаний пространства |
|---|---|---|---|---|
| **Глобальное** | уровень всего канала | сохранённые глобальные чаты | глобальные заметки (с AI-флагом) | профиль канала, системный промпт, помеченные глобальные заметки, метрики канала, при необходимости — все посты |
| **Локальное** | один пост | локальные чаты этого поста | локальные заметки этого поста (с AI-флагом) | сам пост, его медиа, его метрики, его локальные заметки + **наследуется глобальная база знаний** (профиль канала, системный промпт, помеченные глобальные заметки, метрики канала) |

Локальное пространство **не видит** других локальных пространств: ассистент поста не имеет доступа к чужим постам и чужим локальным заметкам. Глобальное пространство, наоборот, видит всю систему целиком.

Главный смысл такой модели — локальный ассистент работает в фокусе конкретного поста, но не «вакуумно»: тон, рубрики, ЦА, запреты и помеченные глобальные заметки автоматически попадают в его контекст из глобального пространства.

---

## Глобальное пространство

Точки входа в UI:

- `home` — стартовый экран, новый пустой глобальный чат (composer в центре).
- `gchat` — открытый сохранённый глобальный чат.
- `chats` → вкладки **«Все»** / **«Глобальные»** / **«Локальные»** — каталог чатов.
- `notes` → scope **«Все»** / **«Глобальные»** / **«Локальные»** — единый каталог заметок.

База знаний глобального пространства шире локального и собирается из:

- содержимого канала (метаданные, статистика на `/analytics`);
- профиля канала (`/profile` → вкладка `Канал` — модель описана в [05-channel-profile.md](./05-channel-profile.md));
- системного промпта и настроек моделей (`/profile` → вкладка `Настройки`);
- подключения и истории Telegram-канала (та же вкладка `Настройки`);
- всех глобальных заметок с флагом «учитывать ИИ»;
- (потенциально) содержимого всех постов и их публичных метрик.

В composer’е глобального чата доступно универсальное attach-меню `+`: прикрепить пост (с подменю списка постов из ленты), прикрепить файл с компьютера, медиа из уже привязанных к сообщению постов. Привязка постов выполняется на уровне конкретного сообщения, отдельной сущности «закреплённые посты чата» в UI нет.

---

## Локальное пространство поста

Открывается из ленты по клику на карточку поста. Внутри пространства поста — четыре режима, переключаются кнопками в шапке (`Заметки`, `Чаты`) и входом в комментарии:

- **`chat`** — открытый локальный чат (или новый пустой).
- **`chats`** — список локальных чатов этого поста (карточки, поиск).
- **`notes`** — сетка локальных заметок этого поста (поиск).
- **`comments`** — комментарии к опубликованному посту (поиск, ответы, добавление).

Каркас, общий для всех режимов:

- сама шапка пространства поста (хлебные крошки `Лента / …` слева; справа `↑ К посту`, кластеры `Заметки` / `Чаты`, `← Назад`, `•••`; поиск в подрежимах);
- стек переходов внутри пространства: переключение режимов и открытие чатов из списка кладёт текущее состояние в стек, кнопка `← Назад` сначала разматывает стек и только потом уходит на ленту.

### Режим `chat`

- Карточка поста (TG-стиль медиа над текстом + текст) всегда первая в скролле.
- Inline-редактирование поста по `✏ Редактировать`: textarea растёт по контенту от исходного размера, медиа остаётся сверху, доступна кнопка `🖼 Добавить медиа`.
- История активного локального чата ниже карточки. При входе в пространство активного чата нет — это пустой «черновик нового чата».
- Composer внизу с собственным attach-меню: `Прикрепить пост`, `Медиа из поста` (медиа текущего поста), `Прикрепить файл с компьютера`, `Медиа из прикреплённых постов` (когда привязаны другие посты).

### Режим `chats`

- Список карточек локальных чатов поста (название чата = первое сообщение, превью, дата).
- Кнопка `+ Новый чат` создаёт пустой черновик нового чата (мгновенно переключает в режим `chat` с `currentPostChatId = null`).
- Клик по карточке → режим `chat` с выбранным `currentPostChatId`.
- Каждая карточка имеет `🗑` для удаления чата.

### Режим `notes`

- Сетка локальных заметок поста (карточки 3 в ряд на десктопе).
- Карточка `+` создаёт новую заметку.
- На карточке — переключатель `● ИИ / ○ ИИ` и `🗑`.
- Клик по карточке открывает страницу заметки.

### Жизненный цикл локального чата

- При входе в пространство поста активного чата нет: `currentPostChatId === null`, composer пуст.
- Первое отправленное сообщение **создаёт** локальный чат (с автозаголовком из первых ~40 символов) и фиксирует его id.
- AI-ответ приходит в этот же чат. Дальше работа идёт в нём.
- Пользователь может в любой момент переключиться на `Чаты`, выбрать другой чат или создать ещё один через `+ Новый чат`.
- Память между разными локальными чатами одного поста и между постами не делится: каждый чат — отдельный контекст для ИИ.

---

## Границы контекста ИИ

- **Локальный чат** видит: сам пост (текст, медиа, метрики, история редактирования), локальные заметки поста с AI-флагом, и **всю наследуемую глобальную базу** (профиль канала, системный промпт, помеченные глобальные заметки, метрики канала). Другие посты и чужие локальные заметки в контекст не попадают. Другие локальные чаты того же поста для него тоже не видны — каждый чат отдельная сессия.
- **Глобальный чат** видит общий контекст канала: профиль, помеченные глобальные заметки, метрики, при необходимости — выборку постов и их локальные данные.
- AI-флаг есть у заметок обоих типов и решает, попадает ли заметка в контекст соответствующего ИИ. Глобальная заметка с включённым AI-флагом автоматически попадает и в локальные пространства всех постов (как часть наследуемой глобальной базы).
