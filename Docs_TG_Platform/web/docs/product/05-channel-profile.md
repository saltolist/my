# Профиль канала

Профиль — **база знаний глобального пространства** (см. [03-spaces.md](./03-spaces.md)). Всё, что заполнено на вкладке `Канал`, попадает в контекст глобального и локального ИИ.

Локальное пространство поста **наследует** эту базу целиком: тон, ЦА, рубрики и правила работают в локальных чатах без отдельной настройки на уровне поста. Своё у поста — только сам пост и его локальные заметки с AI-флагом.

> **UI reference:** [`profile-settings.md`](../ux/components/widgets/profile-settings.md#channeltab) · [`pages.md`](../ux/pages.md#9-profile).  
> Ниже — **только поля, которые есть в интерфейсе**. Отдельных секций «демография», «бренд-бук», «конкуренты» в UI нет — их содержание при необходимости пишут в textarea-полях (например, портрет ЦА — в поле «ЦА»).

## Вкладка «Канал»

Intro: «Канал как база знаний ИИ».

### Ядро канала

| Поле | Назначение |
|---|---|
| **О чём канал** | Тема и фокус канала |
| **Цель** | Зачем канал существует, какую задачу решает |
| **ЦА** | Портрет аудитории — сюда же уместно описать демографию, боли, сегменты **текстом**, без отдельных контролов |
| **Ценность** | Что получает читатель, обещание канала |
| **Портрет автора** | Кто пишет, экспертиза, позиция |

Кнопки **Сохранить** / **Сбросить** при `channelProfileDirty`.

### Голос и формат

| Поле | Назначение |
|---|---|
| **Тон** | Как звучит канал (формальность, юмор, экспертность) |
| **Обращение** | К читателю: «ты» / «вы», от первого лица и т.д. |
| **Базовый формат поста** | Типичная структура, длина, хук, CTA — **одно поле**, не отдельный блок «структурные паттерны» |

### Правила

| Поле | Назначение |
|---|---|
| **Обязательно** | Что ИИ должен соблюдать всегда |
| **Избегать** | Запрещённые темы, слова, форматы |

### Рубрики

Динамический список. **На каждую рубрику — только два поля:**

- **Название**
- **Описание**

Добавление / удаление строк. Отдельный dirty-state `rubricsDirty`. Полей «частота», «стиль», «примеры», «теги» в UI нет.

Footer: «Технические настройки — на вкладке Настройки».

## Вкладка «Настройки»

### Тема

`☀ Светлая` · `🖥 Системная` · `🌙 Тёмная` — сохраняется в localStorage сразу.

### ИИ-движок

- **LLM-модели** — провайдер, модель, API-ключ, «Включить в мультиответ»
- **Web Search модели** — то же
- **Оркестратор** — одна активная модель
- **Web Reasoner** — одна активная модель
- **RAG Reasoner** — одна активная модель
- **Мультиответ** — checkbox + флаги на моделях
- **Системный промпт** — textarea с явным сохранением

Параметров «температура» / «max tokens» в форме нет.

### Telegram

MTProto: api_id, api_hash, телефон, код, выбор канала, сброс настроек.

**Омниканальный бот** — отдельный блок с Bot API token.

Поля `sessionName`, `syncMode` есть в типах/seed, **в форме не рендерятся**.
