# Профиль (`profile`)

Экран разделён на три вкладки: `Канал`, `Настройки`, `Аналитика платформы`.

```
┌─────────────────────────────────────────────────────────────────┐
│  Профиль                                                        │
│ ─────────────────────────────────────────────────────────────── │
│ [Канал] [Настройки] [Аналитика платформы]                       │
└─────────────────────────────────────────────────────────────────┘
```

## Шапка

- Заголовок `Профиль` (`PageHeader`).
- Поиска и `← Назад` нет.

## Вкладка `Канал`

Демо-секции (статичные, без редактора):

- Целевая аудитория;
- Рубрики;
- Тон и голос;
- Запрещённые темы.

Полная модель — в [`../concept/05-channel-profile.md`](../concept/05-channel-profile.md).

## Вкладка `Настройки`

### Тема оформления

Сегментированный переключатель из трёх кнопок: `☀ Светлая`, `🖥 Системная`, `🌙 Тёмная`. Выбор сохраняется в `localStorage` и применяется через атрибут `data-theme` на `<html>`; режим `Системная` следует `prefers-color-scheme`.

### ИИ-движок

- Несколько LLM-моделей: для каждой — выбор провайдера и модели через `ModelPicker` (пилюля с иконкой `🧠` и dropdown’ом), поле API-ключа с явной рамкой, флаг `Включить в мультиответ`, кнопка «сделать активной».
- Несколько Web Search моделей — аналогичный блок с иконкой `🔎`.
- Общий переключатель `Мультиответ` (включается, если у нескольких моделей включён флаг `Включить в мультиответ`).
- Системный промпт — textarea с явной рамкой.

Параметры `температура` и `max tokens` в текущем демо не вынесены в UI.

### Telegram

MTProto-форма:

- `api_id`, `api_hash`;
- телефон;
- код подтверждения;
- `sessionName`;
- выбор `syncMode` через `ModelPicker`;
- кнопки подключения/отключения канала.

Все поля оформлены с явными рамками и фоном.

## Вкладка `Аналитика платформы`

Секции:

- Использование ИИ: запросы, токены, стоимость;
- Активность: чаты, заметки, посты.

## Примечание

В текущем демо вкладка `Аналитика платформы` не включает историю агентских действий и отдельный блок оценок лайк/дизлайк.
