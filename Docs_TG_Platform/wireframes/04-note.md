# Страница заметки (`note`)

Заметка поддерживает два режима: `view` и `edit`.

## Шапка

```
┌─────────────────────────────────────────────────────────────────┐
│ Заметки / Название          или   Лента / Пост / Название       │
│                                     [← Назад] [•••]             │
└─────────────────────────────────────────────────────────────────┘
```

- Слева — хлебные крошки:
  - глобальная: `Заметки / Название`;
  - локальная: `Лента / Название поста / Название` (средняя часть — ссылка на пост).
- Справа: `← Назад`, меню `•••` (`●/○ Учитывать в ИИ`, `Удалить заметку` / `Отменить` для новой).
- Заголовок заметки в теле — textarea с авто-подгонкой шрифта под одну строку (`useFitTitleSize`).

## Режим `view`

Показывает:

- заголовок заметки (textarea, только чтение по смыслу view);
- тулбар: иконка «редактировать», кнопка `Сохранить` (неактивна без изменений);
- для локальной заметки — метка `📌 Локальная` и ссылка `→ пост`;
- отрендеренный текст заметки (embed-чипы файлов/медиа);
- блок `Вложения` (если есть файлы);
- дату создания/изменения внизу.

Переключатель AI-флага — в меню `•••`, не в теле страницы.

## Режим `edit`

Показывает:

- textarea заголовка;
- тулбар: `📎` (файл), иконка «просмотр», `Сохранить` (активна при изменениях);
- для локальной заметки — `📌 Локальная` + `→ пост`;
- редактируемое тело заметки (`NoteBodyEditor`);
- блок `Вложения` с drag-and-drop.

## Drag-and-drop в заметке

- Файл из панели `Вложения` можно перетащить в тело — появится embed-чип `[имя файла]`.
- Embed-чипы в теле можно перетаскивать для изменения порядка (только в режиме `edit`).
- Внешние файлы с рабочего стола при drop в тело также добавляются как вложения.

## Поведение переходов

- При уходе со страницы с несохранёнными изменениями — подтверждение (`window.beforeunload`).
- `Сохранить` активна только при наличии изменений относительно последнего сохранения.
