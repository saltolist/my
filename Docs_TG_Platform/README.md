# TG Platform

Open source, self-hosted платформа для автора Telegram-канала: единое рабочее пространство, где связаны посты, заметки, аналитика и контекстный ИИ.

## Зачем это нужно

Обычный процесс ведения канала разорван между инструментами: идеи в заметках, публикации в одном сервисе, аналитика в другом, ИИ в третьем — и почти всегда без знания контекста канала.

TG Platform решает это как цельную систему:

- один интерфейс для полного цикла `идея -> черновик -> публикация -> анализ`
- ИИ, который знает аудиторию, рубрики, стиль и историю контента
- прозрачный контроль над данными и выбором ИИ-движка

## Что уже спроектировано

- **Лента постов с тремя статусами**: опубликованные, отложенные, черновики
- **Пространство поста**: чат с локальным ИИ, inline-редактирование и локальные заметки
- **Глобальный чат (глобальный ИИ)**: стратегия на уровне всего канала, с возможностью привязывать к сообщению посты и их медиа
- **Система заметок**: локальные и глобальные, с управлением попаданием в контекст ИИ
- **Профиль канала**: ЦА, рубрики, тон, ограничения, референсы
- **Две аналитики**:
  - аналитика канала (контент и рост)
  - аналитика платформы/ИИ (токены, стоимость, активность, действия агентов)

## Ключевые принципы

- **Контекст - главный актив.** Качество ответов ИИ напрямую зависит от полноты профиля и заметок.
- **Пост - живое пространство.** После публикации пост остается рабочим объектом для анализа и доработок.
- **ИИ - соавтор, а не замена.** Все решения остаются за автором канала.
- **Self-hosted и открытость.** Инфраструктура и данные под контролем пользователя.

## Архитектура ИИ

- **Локальный ИИ (уровень поста)**: работает в рамках конкретного поста, его метрик, истории и локальных заметок.
- **Глобальный ИИ (уровень канала)**: работает в масштабе всей системы — посты, профиль, глобальные заметки, аналитика.
- **Агентские действия по команде**: публикация, редактирование, создание черновиков и другие действия происходят явно по запросу пользователя.

## Навигационная модель платформы

```
TG Platform
├── Логотип / [+ Чат]           -> стартовый экран (глобальный чат)
├── Лента                        -> все посты
│   └── Пространство поста       -> чат + заметки конкретного поста
├── Чаты                         -> история локальных и глобальных чатов
├── Заметки                      -> каталог заметок (глобальные + локальные)
├── Аналитика                    -> метрики канала
└── Профиль
    ├── Канал                    -> контекст для ИИ
    ├── Настройки                -> модель, API-ключ, Telegram
    └── Аналитика платформы      -> токены, стоимость, действия агентов
```

## Статус проекта

Сейчас это **продуктовая документация и UX-проектирование**: концепт, сценарии, структура страниц и wireframes.

Кодовая реализация платформы находится в следующем этапе, приоритетно по roadmap:

1. Основа платформы (лента + интеграция Telegram)
2. Пространство поста и локальный ИИ
3. Профиль и контекст
4. Глобальный ИИ и полный цикл публикации

## Кому подойдет

- авторам Telegram-каналов, которые ведут контент самостоятельно
- тем, кто хочет уйти от SaaS-зависимости и держать данные у себя
- тем, кому нужен ИИ с долгой памятью о канале, а не универсальный чат

## Лицензия и вклад

Проект задуман как open source. Перед началом разработки можно открыть issue с предложениями по архитектуре, UX и приоритетам roadmap.
