# Модули платформы (актуально для demo)

Документ фиксирует модули в текущем демо. Это рабочая карта интерфейса, а не целевая спецификация будущих версий.

## Структура навигации

```
TG Platform
├── [✦ лого] / [Глобальный чат]  ← home (новый глобальный чат); панель сворачивается в rail
├── Лента                        ← все посты: опубликованные / отложенные / черновики
│   └── Пространство поста       ← локальные чаты + заметки + комментарии поста
│       ├── режим chat           ← открытый локальный чат
│       ├── режим chats          ← список локальных чатов (поиск)
│       ├── режим notes          ← сетка локальных заметок (поиск)
│       ├── режим comments       ← комментарии (только опубликованные; поиск)
│       └── Страница заметки     ← локальная заметка
├── Аналитика канала             ← сводка по каналу
├── Заметки ▾                    ← каталог + недавние в сайдбаре
│   └── Страница заметки         ← глобальная или локальная
├── Чаты ▾                       ← каталог + недавние в сайдбаре
└── Профиль канала
    ├── Канал                    ← базовый контекст
    ├── Настройки                ← ИИ-движок, тема оформления, Telegram
    └── Аналитика платформы      ← usage-метрики
```

---

## Глобальный чат

Глобальный чат состоит из двух экранов:

- `home` — новый чат (пустое состояние),
- `gchat` — уже созданный чат.

**Доступно в демо:**

- отправка сообщений;
- сохранение чатов в разделе `Чаты`;
- удаление чата из меню `•••`;
- composer с выбором LLM-модели, Web Search-модели и режимом мультиответа;
- attach-меню `+` со следующими действиями:
  - `Прикрепить пост` — выбор поста из ленты, привязывается к сообщению как «чип» (синтаксис `@<title>;` делает то же самое);
  - `Медиа из прикреплённых постов` — миниатюры медиа только из уже привязанных к сообщению постов;
  - `Прикрепить файл с компьютера`.

**Ограничение демо:** отдельной сущности «закреплённые посты чата» в UI больше нет — всё прикрепление выполняется на уровне конкретного сообщения через composer.

---

## Лента

Лента показывает посты в фиксированном порядке секций:

1. `Опубликованные`
2. `Отложенные`
3. `Черновики`

**Что работает:**

- создание нового черновика через поле ввода внизу (в composer’е ленты `+` сразу открывает выбор файла; к одному черновику можно прикрепить несколько медиа);
- поиск по постам в шапке;
- переход в пространство поста по клику на карточку;
- отображение статуса, даты, рубрики; у опубликованных — пилюли реакций, просмотры, репосты, счётчик комментариев (клик → режим `comments` поста); TG-стиль медиа-блок над текстом карточки;
- drag-handle `⠿` у черновиков для drag-and-drop-перестановки в пределах секции.

**Замечание:** drag-and-drop в демо — это часть фронт-поведения (см. [12-demo.md](./12-demo.md)); порядок черновиков сохраняется в памяти вкладки и сбрасывается при перезагрузке.

## Пространство поста

Локальное пространство конкретного поста. Содержит четыре режима — `chat`, `chats`, `notes`, `comments` — и общий каркас (шапка + карточка поста).

**Шапка:**

- хлебные крошки `Лента / …` слева (до трёх уровней при открытом чате или подрежиме);
- справа: `↑ К посту`, кластеры `Заметки` / `Чаты` (с `+ Новая заметка` / `+ Новый чат` в активном режиме), `← Назад`, меню `•••`;
- поиск по центру в режимах `chats`, `notes`, `comments`.

При переключении режимов и при открытии конкретного локального чата текущее состояние кладётся в стек. Кнопка `← Назад` сначала разматывает этот стек (возвращает на предыдущий режим/чат внутри поста) и только при пустом стеке уходит на ленту.

**Меню `•••`:**

- `📝 Новый чат` — создаёт пустой черновик нового локального чата и переключает в режим `chat`;
- `📢 Опубликовать` (для черновиков и отложенных);
- `🕐 Запланировать` (для черновиков);
- `✕ Отменить публикацию` (для отложенных);
- `🗑 Удалить пост`.

**Режим `chat`:**

- первое сообщение в скролле = карточка поста (TG-стиль медиа над текстом + текст);
- inline-редактирование (`✏ Редактировать` + `Отмена/Сохранить`): textarea авто-ресайзится от исходного размера, медиа остаётся сверху, есть кнопка `🖼 Добавить медиа`;
- ниже — история активного локального чата; при входе в пространство активного чата нет (composer пуст);
- первое отправленное сообщение создаёт локальный чат (заголовок берётся из ~40 первых символов), AI-ответ приходит в него;
- composer внизу: `+` открывает меню с `Прикрепить пост`, `Медиа из поста`, `Прикрепить файл с компьютера` и `Медиа из прикреплённых постов` (когда привязаны другие посты).

**Режим `chats`:**

- список карточек локальных чатов поста (название = первое сообщение, превью, дата);
- кнопка `+ Новый чат` создаёт пустой черновик нового чата;
- клик по карточке открывает соответствующий чат в режиме `chat`;
- `🗑` удаляет локальный чат.

**Режим `notes`:**

- сетка локальных заметок поста;
- поиск по заголовку и тексту;
- создание новой заметки;
- переключение AI-флага и удаление заметки.

**Режим `comments`** (только опубликованные посты):

- список комментариев с поиском, ответы с цитатой, добавление через composer внизу;
- вход из ленты (`💬` в метриках) или со строки «Комментарии» на карточке поста.

Память между локальными чатами одного поста не делится — каждый чат отдельный контекст для ИИ. В чатах доступно редактирование пользовательских сообщений с ветками и toolbar у ответов ИИ.

## Чаты

Раздел содержит две вкладки:

- `Глобальные` — сохранённые глобальные чаты;
- `Локальные` — все локальные чаты со всех постов; каждая карточка — отдельный локальный чат, а не агрегат по посту (один пост может занимать несколько карточек, по числу его чатов).

На карточке локального чата отображается название связанного поста, превью последнего сообщения, дата и кнопка `→ пост`. Клик по карточке открывает соответствующий локальный чат в пространстве поста (без сброса `currentPostChatId`).

---

## Заметки

Раздел заметок в демо объединяет два scope:

- `Глобальные`,
- `Локальные`.

Есть:

- поиск по заголовку и тексту;
- фильтр `Все / В контексте ИИ / Не в контексте`;
- переключение AI-флага на карточке;
- удаление;
- переход на страницу заметки.

---

## Аналитика канала

Доступна отдельным экраном в навигации.

В демо показывает:

- период (`7/30/90/Всё время`);
- сводные карточки (`Охваты`, `Подписчики`, `ER`);
- блок `По рубрикам`;
- таблицу `Лучшие посты за период` с переходом к посту.

---

## Профиль

Три вкладки:

- `Канал` — ЦА, рубрики, тон и голос, запрещённые темы (в демо вкладка показана упрощённо, как статичный демо-профиль; полная модель — в [05-channel-profile.md](./05-channel-profile.md));
- `Настройки` — переключатель темы оформления (`☀ Светлая` / `🖥 Системная` / `🌙 Тёмная`, выбор сохраняется в `localStorage`), несколько LLM-моделей и несколько Web Search моделей с флагом «Включить в мультиответ», переключатель `Мультиответ`, системный промпт, MTProto-форма Telegram (api_id, api_hash, телефон, код, sessionName, syncMode, подключение канала). Провайдеры и режимы выбираются через тот же `ModelPicker`, что и в composer’е; поля API-ключей/промпта/Telegram оформлены с явными рамками. Параметры `температура` и `max tokens` пока не вынесены в UI;
- `Аналитика платформы` — запросы, токены, стоимость, общая активность.
