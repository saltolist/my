# ИИ-система

## Двухуровневая архитектура

Платформа использует два типа ИИ-помощников, работающих на разных уровнях.

---

## Локальный ИИ (уровень поста)

Каждое пространство поста содержит свои **локальные чаты** — отдельные переписки с контекстным помощником, привязанным к этому посту. Чатов у поста может быть произвольно много, но память между ними не делится: каждый чат — самостоятельная сессия с одним и тем же набором базовых знаний.

В демо это выглядит так: при заходе в пространство поста активного чата нет (composer пуст). Первое отправленное сообщение создаёт новый локальный чат, в котором живёт дальнейшая переписка. Список существующих чатов поста доступен через режим `Чаты` в шапке пространства поста; новый чат запускается тем же composer’ом после нажатия `+ Новый чат` или пункта `Новый чат` в меню `•••`.

**Контекст локального ИИ (каждого отдельного чата) = своя локальная база + наследуемая глобальная:**

Своё, локальное:
- Содержание поста (текст, медиа, ссылки)
- Метрики поста (охваты, реакции, репосты)
- История редактирования
- Локальные заметки, прикреплённые к посту (с AI-флагом)
- История сообщений **этого конкретного локального чата** (другие локальные чаты того же поста не учитываются)

Наследуется из глобального пространства:
- Профиль канала целиком: ЦА, рубрики, тон и голос, запрещённые темы, бренд-бук
- Системный промпт уровня канала
- Глобальные заметки с включённым AI-флагом
- Метрики канала

Чужие посты и чужие локальные заметки в контекст локального чата **не попадают** — локальные пространства друг друга не видят (см. [03-spaces.md](./03-spaces.md)).

**Что умеет:**
- Доработать, переписать или расширить пост
- Предложить заголовки и подписи к медиа
- Создать тред или продолжение темы
- Объяснить, почему пост сработал или нет
- Предложить форматы для этой же темы

**Агентские действия локального ИИ:**

Локальный ИИ может совершать действия по прямой команде пользователя в чате. Каждое действие — это запрос → действие → чёткий результат. Автономной работы без команды нет.

| Команда пользователя | Действие агента | Результат |
|---|---|---|
| "Отредактируй пост: убери последний абзац" | Редактирует текст поста | Пост обновлён, изменение видно в шапке |
| "Опубликуй этот пост" | Публикует черновик в канал | Статус поста меняется на "опубликован" |

## Глобальный ИИ (уровень канала)

Глобальный помощник в отдельном чате. Видит весь канал целиком и работает как стратегический советник.

**Контекст глобального ИИ:**
- Все посты канала (или релевантная выборка через MemPalace)
- Профиль канала полностью
- Глобальные заметки (или помеченные флагом "в контекст")
- Метрики канала
- Запиненные посты — передаются с акцентом как приоритетный контекст

**Запин постов.**
Пользователь может добавить один или несколько конкретных постов в контекст текущего чата отдельной кнопкой. Запиненные посты учитываются в первую очередь — это позволяет фокусировать глобальный ИИ на нужных материалах на фоне всего канала.

**Что умеет:**
- Составить контент-план на неделю/месяц
- Проанализировать, какие рубрики работают
- Дать рекомендации по стратегии роста
- Сравнить несколько постов и объяснить разницу в результатах
- Найти паттерны в успешных и неуспешных постах

**Агентские действия глобального ИИ:**

| Команда пользователя | Действие агента | Результат |
|---|---|---|
| "Создай черновик поста на тему X" | Создаёт новый пост-черновик | Черновик появляется в ленте |
| "Опубликуй пост [название]" | Публикует указанный черновик | Статус поста меняется на "опубликован" |

---

## Принцип работы агентов

Агенты работают только по прямой команде пользователя. Никакой автономной фоновой активности нет.

**Цикл:** команда пользователя → действие агента → чёткий видимый результат.

Пользователь всегда знает, что именно сделал агент, и может это проверить. История агентских действий хранится в технических настройках профиля.

## Медиа в чате

В чат (как локального, так и глобального уровня) можно добавить медиафайлы:
- **Из поста** — прикрепить медиа, уже привязанное к посту (только в локальном чате)
- **С компьютера** — загрузить файл напрямую в чат

ИИ видит прикреплённые медиа и может работать с ними: описать, предложить подпись, проанализировать в контексте поста.

## Контекст ИИ

Качество ответов прямо зависит от заполненности профиля и заметок. Контекст собирается из нескольких источников:

| Источник | Что даёт ИИ | Доступен глобальному | Доступен локальному |
|---|---|---|---|
| Профиль канала | Тон, стиль, ЦА, рубрики, запреты | да | да (наследуется) |
| Системный промпт | Базовые инструкции уровня канала | да | да (наследуется) |
| Глобальные заметки с AI-флагом | Идеи, исследования, база знаний | да | да (наследуется) |
| Метрики канала | Какой контент получает охваты | да | да (наследуется) |
| История постов / выборка постов | Паттерны, частота, что работало | да | — |
| Запиненные посты | Приоритетный контекст по выбору пользователя | да | — |
| Конкретный пост (текст, медиа, метрики) | Содержание и история поста | через привязку поста к сообщению | да (как «своя сущность») |
| Локальные заметки поста с AI-флагом | Заметки, привязанные к конкретному посту | — | да |
| История чата | Текущая переписка | да (своего чата) | да (своего чата) |

Локальные пространства друг друга не видят: ассистент поста A не имеет доступа к содержимому поста B и его локальным заметкам.

## Настройка ИИ-движка

Платформа не привязана к конкретной модели. Пользователь выбирает в технических настройках:

- **Модель** — любая совместимая через OpenAI-совместимый API
- **API-ключ** — свой ключ, хранится локально
- **Системный промпт** — возможность дополнить базовый промпт
- **Параметры** — температура, максимальная длина ответа

Можно подключить OpenAI, Anthropic, Mistral, локальную модель через Ollama и любой другой совместимый провайдер.
