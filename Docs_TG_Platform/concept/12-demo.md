# Демо-версия

Демо — это публичная статическая версия интерфейса TG Platform, которую любой пользователь может открыть в браузере и попробовать руками без установки чего-либо локально.

Главная задача демо — показать продуктовую идею (пространства, ленту, локальный и глобальный ИИ, заметки, профиль). Пользователь свободно ходит по интерфейсу, всё фронтовое поведение работает, отсутствует только бэкенд: реальная интеграция с Telegram и реальный вызов LLM заменены на заскриптованные ответы и заглушки.

---

## Зачем нужна демо-версия

- Дать оценщику (потенциальный пользователь, контрибьютор, читатель README) сразу пощупать продукт.
- Не требовать установки Telegram-интеграции, API-ключей LLM, сервера, базы данных.
- Зафиксировать единое визуальное и поведенческое описание продукта рядом с концептом и wireframes.
- Снизить порог входа в обсуждение: вместо текста и схем — живой интерфейс.

Демо не заменяет полную реализацию (см. [08-roadmap.md](./08-roadmap.md)). Это витрина продуктового UX.

---

## Где живёт демо

- Исходники: `Docs_TG_Platform/demo/` в этом репозитории.
- Стек: Next.js (App Router) + React + TypeScript, статический экспорт (`output: "export"`).
- Публикация: GitHub Pages этого же репозитория, готовая статика лежит в `Docs_TG_Platform/demo/out/` после `npm run build`.
- Доступ: одна ссылка, открывается в любом современном браузере.

Технические требования к демо:

- статический экспорт — никакого Node-сервера в проде;
- без сетевых вызовов к LLM и Telegram;
- всё состояние держится в памяти вкладки;
- стили — один глобальный `app/globals.css`, без CSS-in-JS и Tailwind, чтобы визуальное соответствие проектируемому интерфейсу было гарантированным.

## Что показывает демо

Демо отражает текущую модель платформы:

- лента постов с тремя секциями (`Опубликованные / Отложенные / Черновики`) и поиском по постам;
- пространство поста с четырьмя режимами (`chat` + `chats` + `notes` + `comments`) и стеком переходов внутри поста;
- комментарии к опубликованным постам (просмотр, поиск, ответ, добавление);
- сайдбар с недавними чатами/заметками и сворачиваемым rail;
- глобальный чат (`home` + `gchat`);
- список чатов (`Все` / `Глобальные` / `Локальные` — по карточке на каждый локальный чат) с поиском;
- каталог заметок (scope `Все` / `Глобальные` / `Локальные`, AI-флаг у обоих типов, поиск, фильтр по контексту ИИ);
- аналитика канала;
- профиль (`Канал` / `Настройки` / `Аналитика платформы`) с редактируемой моделью канала и переключателем темы.

Всё интерактивное поведение фронта работает полноценно, в том числе:

- drag-and-drop черновиков в ленте (рель с трёмя точками слева; перестановка с сохранением порядка в памяти вкладки; автоскролл ленты у краёв экрана);
- drag-and-drop в заметках (файлы из панели вложений в тело; перестановка embed-чипов);
- переключатель ширины карточек поста в ленте (`500` / `390` / `270` px);
- inline-редактирование текста и медиа поста в локальном чате (textarea авто-ресайзится, медиа сверху, кнопка `🖼 Добавить медиа`);
- множественные локальные чаты в пространстве поста: при входе активного чата нет, первое сообщение создаёт чат; список чатов доступен через режим `Чаты` в шапке; новый чат — через `+ Новый чат` или пункт меню `•••`;
- переключение режимов `chat` / `chats` / `notes` / `comments` в пространстве поста; стек переходов, `← Назад` сначала разматывает стек и только потом уходит на ленту;
- редактирование пользовательских сообщений в чатах с ветками; toolbar у ответов ИИ;
- AI-флаг и фильтрация заметок;
- меню `•••` со всеми действиями (новый чат, публикация, планирование, удаление и т.п.);
- composer с выбором LLM/Web Search моделей и режимом мультиответа, авто-ресайзом textarea (до 10 строк на `home`, до 16 в чатах и ленте);
- универсальное attach-меню `+`: прикрепить пост (плюс синтаксис `@<title>;` с inline-чипом в `contenteditable`), медиа из поста (на `post`), медиа из прикреплённых к сообщению постов, файл с компьютера; в `feed` `+` сразу открывает file picker;
- единая шапка (`PageHeader`) в стиле liquid glass: заголовок или крошки слева, опциональный поиск по центру, `↑ К посту` / переключатели режимов / `← Назад` / `•••` справа (на `post` — кластеры `Заметки` и `Чаты` с кнопками создания);
- переключатель темы (`☀ Светлая` / `🖥 Системная` / `🌙 Тёмная`) на вкладке `Настройки`, сохраняется в `localStorage`.

Детальное описание экранов — в [10-pages.md](./10-pages.md).

## Чего демо не делает

Демо обрезано ровно на границе бэкенда. Всё, что работает в браузере, доступно пользователю; всё, что требует серверной или внешней интеграции — заменено заглушкой.

- Не публикует реальные посты в Telegram (кнопки публикации меняют статус только локально, в памяти вкладки).
- Не вызывает реальные LLM-модели (ответы ИИ — заранее заготовленные строки).
- Не сохраняет состояние между сессиями (перезагрузка страницы возвращает демо к стартовому состоянию).
- Не имитирует ошибки сети, лимиты токенов, задержки реального API.
- Не реализует продакшен-фичи, связанные с сервером: реальные права доступа, аутентификация, синхронизация.

## Стартовое состояние

Демо открывается уже «обжитым», а не пустым:

- профиль канала заполнен полной моделью из документации (ЦА, рубрики, тон, структура, запреты, бренд-бук, конкуренты и референсы);
- в ленте есть посты во всех трёх секциях (`Опубликованные / Отложенные / Черновики`); у опубликованных — реакции, метрики и демо-комментарии;
- есть глобальные и локальные заметки с проставленным AI-флагом;
- есть несколько сохранённых глобальных и локальных чатов;
- в настройках выбраны мок-модели LLM и Web Search;
- аналитика показывает реалистичные мок-метрики.

Это даёт пользователю сразу богатую среду, чтобы пощупать продукт без прохождения «онбординга».

## Заскриптованные ответы ИИ

В демо нет вызова реальных LLM. Ответы подставляются по ключевым словам в тексте сообщения (`lib/replies.ts`).

### Глобальный ИИ (`getGlobalReply`)

| Ключевое слово в сообщении | Тема ответа |
|---|---|
| `контент-план` | план публикаций на неделю |
| `рубрик` | сравнение рубрик по охватам |
| `тем` | идеи постов по профилю канала |
| `охват` | динамика охватов |

Если ни одно ключевое слово не совпало — нейтральный уточняющий ответ («дай больше деталей…»).

### Локальный ИИ (`getPostReply`)

| Ключевое слово | Тема ответа |
|---|---|
| `перепиш` / `rewrite` | переработка вступления |
| `заголов` | варианты заголовков |
| `почему` / `аналити` | аналитика по паттернам канала |

Иначе — общий уточняющий ответ по контексту поста.

### Другие заглушки

- после **редактирования** пользовательского сообщения в чате хвост ветки заменяется фиксированным ответом `STUB_REPLY_AFTER_USER_EDIT` (`lib/chatPaths.ts`);
- плейсхолдер composer’а по умолчанию: `Сообщение... введите @ чтобы прикрепить пост` (без per-chat сценарных подсказок).

Поведение предсказуемо: один и тот же ввод даёт один и тот же ответ в рамках сессии.

## Архитектура демо

Данные и логика разделены:

- мок-база (профиль, посты с `comments`, заметки, чаты, настройки) — `lib/data.ts`;
- ответы ИИ — `lib/replies.ts` (`getGlobalReply`, `getPostReply`);
- ветки и правки сообщений — `lib/chatPaths.ts`;
- стейт — `state/AppContext.tsx` (actions: `sendHome`, `sendGChat`, `sendPost`, `openPost`, `openPostComments`, …);
- UI: `Composer`, `AttachMenu`, `ModelPicker`, `PostMediaBlock`, `PostCommentsPanel`, `PageHeader`, `Sidebar`.

## Ограничения и предупреждения

Чтобы демо не воспринималось как продукт:

- в README демо и в meta description (`layout.tsx`) отмечено, что это статическая витрина без реальной интеграции; на экране `home` отдельной плашки нет;
- все «опасные» действия (публикация, удаление канала, оплата) ничего не делают за пределами вкладки;
- ответы ИИ — заранее заготовленные строки, а не вывод модели; после правки сообщения подставляется явная заглушка с пометкой «отредактировано».

---

## Когда демо считается готовой

Минимальный критерий готовности демо как раздела документации:

- все экраны из [10-pages.md](./10-pages.md) доступны и проходимы;
- стартовое состояние (профиль + посты + заметки + чаты) загружено и согласовано с [02-modules.md](./02-modules.md) и [10-pages.md](./10-pages.md);
- в `lib/replies.ts` есть осмысленные ответы на характерные ключевые слова (глобальный и локальный ИИ);
- свободный ввод в чатах не ломает интерфейс и получает нейтральный fallback-ответ;
- ссылка на демо опубликована в `README.md` репозитория;
- README демо описывает структуру файлов и сценарный слой.
