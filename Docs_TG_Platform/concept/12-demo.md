# Демо-версия

Демо — это публичная статическая версия интерфейса TG Platform, которую любой пользователь может открыть в браузере и попробовать руками без установки чего-либо локально.

Главная задача демо — показать продуктовую идею (пространства, ленту, локальный и глобальный ИИ, заметки, профиль). Пользователь свободно ходит по интерфейсу, всё фронтовое поведение работает, отсутствует только бэкенд: реальная интеграция с Telegram и реальный вызов LLM заменены на заскриптованные ответы и заглушки.

---

## Зачем нужна демо-версия

- Дать оценщику (потенциальный пользователь, контрибьютор, читатель README) сразу пощупать продукт.
- Не требовать установки Telegram-интеграции, API-ключей LLM, сервера, базы данных.
- Зафиксировать единое визуальное и поведенческое описание продукта рядом с концептом и wireframes.
- Снизить порог входа в обсуждение: вместо текста и схем — живой интерфейс.

Демо не заменяет полную реализацию (см. [08-roadmap.md](./08-roadmap.md)). Это витрина продуктового UX.

---

## Где живёт демо

- Исходники: `demo/` в этом репозитории.
- Стек: Next.js (App Router) + React + TypeScript, статический экспорт (`output: "export"`).
- Публикация: GitHub Pages этого же репозитория, готовая статика лежит в `demo/out/` после `npm run build`.
- Доступ: одна ссылка, открывается в любом современном браузере.

Технические требования к демо:

- статический экспорт — никакого Node-сервера в проде;
- без сетевых вызовов к LLM и Telegram;
- всё состояние держится в памяти вкладки;
- стили — один глобальный `app/globals.css`, без CSS-in-JS и Tailwind, чтобы визуальное соответствие проектируемому интерфейсу было гарантированным.

## Что показывает демо

Демо отражает текущую модель платформы:

- лента постов с тремя секциями (`Опубликованные / Отложенные / Черновики`) и поиском по постам;
- пространство поста (`chat` + `notes`);
- глобальный чат (`home` + `gchat`);
- список чатов (`Глобальные` / `Локальные`) с поиском;
- каталог заметок (scope + AI-флаг + поиск + фильтр по контексту ИИ);
- аналитика канала;
- профиль (`Канал` / `Настройки` / `Аналитика платформы`).

Всё интерактивное поведение фронта работает полноценно, в том числе:

- drag-and-drop черновиков в ленте (перестановка карточек с сохранением порядка в памяти вкладки);
- inline-редактирование текста и медиа поста в локальном чате (textarea авто-ресайзится, медиа сверху, кнопка `🖼 Добавить медиа`);
- переключение режимов `chat` / `notes` в пространстве поста;
- AI-флаг и фильтрация заметок;
- меню `•••` со всеми действиями (публикация, планирование, удаление и т.п.);
- composer с выбором LLM/Web Search моделей и режимом мультиответа, авто-ресайзом textarea (до 10 строк на `home`, до 16 в чатах и ленте);
- универсальное attach-меню `+`: прикрепить пост, медиа из поста (на `post`), медиа из прикреплённых к сообщению постов, файл с компьютера; в `feed` `+` сразу открывает file picker;
- единая шапка (`PageHeader`) на всех экранах: заголовок или хлебные крошки слева, опциональный поиск по центру (на `feed` — по колонке постов), `← Назад` и `•••` справа.

Детальное описание экранов — в [10-pages.md](./10-pages.md).

## Чего демо не делает

Демо обрезано ровно на границе бэкенда. Всё, что работает в браузере, доступно пользователю; всё, что требует серверной или внешней интеграции — заменено заглушкой.

- Не публикует реальные посты в Telegram (кнопки публикации меняют статус только локально, в памяти вкладки).
- Не вызывает реальные LLM-модели (ответы ИИ — заранее заготовленные строки).
- Не сохраняет состояние между сессиями (перезагрузка страницы возвращает демо к стартовому состоянию).
- Не имитирует ошибки сети, лимиты токенов, задержки реального API.
- Не реализует продакшен-фичи, связанные с сервером: реальные права доступа, аутентификация, синхронизация.

## Стартовое состояние

Демо открывается уже «обжитым», а не пустым:

- профиль канала заполнен (ЦА, рубрики, тон, запрещённые темы) — такой же демо-профиль, как сейчас в `demo/`;
- в ленте есть посты во всех трёх секциях (`Опубликованные / Отложенные / Черновики`);
- есть глобальные и локальные заметки с проставленным AI-флагом;
- есть несколько сохранённых глобальных и локальных чатов;
- в настройках выбраны мок-модели LLM и Web Search;
- аналитика показывает реалистичные мок-метрики.

Это даёт пользователю сразу богатую среду, чтобы пощупать продукт без прохождения «онбординга».

## Сценарии как подсказки в полях ввода

Сценарии в этом демо — это не «прогон по актам», а тонкий сценарный слой поверх свободного использования.

Базовая механика:

- в каждом поле ввода чата (главная страница, открытые глобальные чаты, локальные чаты постов) есть **свободный ввод** — пользователь может написать что угодно;
- в плейсхолдере поля показан **подготовленный вариант сообщения**, по которому в этом конкретном чате есть сценарный ответ ИИ;
- если пользователь отправляет именно подсказанный вариант (или один из вариантов, если их несколько), ИИ отвечает по сценарию — осмысленным заскриптованным сообщением, согласованным с профилем канала и контекстом чата;
- если пользователь отправляет любой другой текст, ИИ отвечает универсальной **заглушкой**, и плейсхолдер с подсказкой в этом чате пропадает.

Таким образом:

- демонстрационная ценность сохраняется: первый «правильный» сценарный путь всегда доступен;
- свобода пользователя не блокируется: можно вводить любой текст, интерфейс не отваливается;
- момент перехода в «свободный режим» виден явно — исчезновение плейсхолдера сигнализирует, что дальше ответы будут заглушками.

### Правила работы подсказок

- Плейсхолдер уникален для конкретного чата (главная, конкретный глобальный чат, конкретный локальный чат поста), а не один на всё демо.
- Один чат может последовательно проводить по нескольким репликам: после успешной сценарной реплики плейсхолдер обновляется на следующий шаг сценария.
- После любого сообщения вне сценария плейсхолдер этого чата гаснет до конца сессии и больше не возвращается.
- Сценарий каждого чата самодостаточен — нет жёсткой последовательности между разными чатами, пользователь может зайти в любой и попробовать.

### Заглушки

- Заглушка — короткое нейтральное сообщение ИИ, явно помеченное как демо-ответ (например: «Это демо-режим, реальный ответ ИИ здесь будет сгенерирован по контексту канала»).
- Заглушка не имитирует уверенный ответ модели и не вводит пользователя в заблуждение.
- Заглушка одинаково работает в глобальном и локальном чатах, разница только в подписи «глобальный / локальный».

## Архитектура сценарного слоя

Сценарные данные отделяются от движка демо:

- общая мок-база (профиль, посты, заметки, чаты, настройки) — в `lib/data.ts`;
- сценарные ответы ИИ и заглушки — в `lib/replies.ts` (`getGlobalReply`, `getPostReply`);
- единый стейт-провайдер `state/AppContext.tsx` оборачивает всё приложение и выдаёт компонентам экранов готовые actions (`sendHome`, `sendGChat`, `sendPost`, `openPost`, `openGChat`, и т. п.);
- composer и attach-меню реализованы как переиспользуемые компоненты (`components/composer/Composer.tsx`, `AttachMenu.tsx`, `ModelPicker.tsx`); медиа-блок поста — `components/post/PostMediaBlock.tsx`; единая шапка — `components/PageHeader.tsx`;
- UI-компоненты не знают о конкретном содержимом сценария, они только спрашивают у сценарного слоя:
  - какой плейсхолдер показать в этом чате сейчас,
  - есть ли сценарный ответ на отправленный текст,
  - если нет — отдать заглушку и погасить плейсхолдер.

Это позволяет:

- править сценарии и подсказки, не трогая UI;
- добавлять новые подсказки и ответы по мере роста демо;
- держать поведение демо предсказуемым: один и тот же ввод даёт один и тот же ответ.

## Ограничения и предупреждения

Чтобы демо не воспринималось как продукт:

- на первом экране и в README демо отмечено, что это статическая витрина без реальной интеграции;
- все «опасные» действия (публикация, удаление канала, оплата) ничего не делают за пределами вкладки;
- сценарные ответы ИИ и заглушки явно помечены как демо-ответы, чтобы пользователь не воспринимал их как реальный вывод модели;
- исчезновение плейсхолдера с подсказкой — это сигнал пользователю, что он вышел за рамки заскриптованных ответов, а не баг.

## Когда демо считается готовой

Минимальный критерий готовности демо как раздела документации:

- все экраны из [10-pages.md](./10-pages.md) доступны и проходимы;
- стартовое состояние (профиль + посты + заметки + чаты) загружено и согласовано с [02-modules.md](./02-modules.md) и [10-pages.md](./10-pages.md);
- в каждом «живом» чате (главная, глобальные чаты, локальные чаты постов) есть хотя бы одна сценарная подсказка с осмысленным ответом ИИ;
- свободный ввод корректно отдаёт заглушку и гасит плейсхолдер;
- ссылка на демо опубликована в `README.md` репозитория;
- README демо описывает структуру файлов и сценарный слой.
