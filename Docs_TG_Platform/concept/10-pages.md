# Структура страниц (актуально для demo)

Документ описывает страницы в текущем демо (`demo/`, Next.js + TS). Цель — зафиксировать, что именно сейчас видит пользователь.

---

## Навигация (левая панель)

Постоянна на всех экранах. На узкой ширине — `MobileTopbar` и выдвижная панель.

**Бренд и сворачивание**

- `✦ TG Platform` (иконка + название) → `home`.
- На ширине ≥761px: кнопка сворачивания панели в узкий rail (только иконки). Состояние в `localStorage` (`tg-demo-sidebar-collapsed`). Разворот — клик по логотипу в свёрнутом rail.

**Пункты (сверху вниз)**

| Пункт | Экран | Дополнительно |
|---|---|---|
| `Глобальный чат` | `home` | старт нового глобального чата |
| `Лента` | `feed` | |
| `Аналитика` | `analytics` | |
| `Заметки` | `notes` | chevron ▾ — список недавних заметок (до 14) |
| `Чаты` | `chats` | chevron ▾ — недавние чаты |
| `Профиль` | `profile` | внизу панели |

**Недавние списки** под `Заметки` и `Чаты`:

- сортировка по активности (длина истории чата / объём текста заметки);
- смешанный список глобальных и локальных элементов;
- в пространстве поста — секции **Этот пост** и **Остальные**;
- клик открывает заметку или чат; у строки — меню `⋯`.

Экран `Чаты` — полный каталог; раскрывающийся список в сайдбаре — быстрый доступ к недавним.

## Общая шапка (`PageHeader`)

На всех верхнеуровневых экранах (`feed`, `chats`, `notes`, `analytics`, `profile`) и на детальных (`gchat`, `post`, `note`) используется единый компонент шапки с фиксированной высотой:

- слева — заголовок (или хлебные крошки `Лента / Название` на `post`, `Заметки / Название` на `note`, `Чаты / Название` на `gchat`);
- по центру — опциональный поиск (есть в `feed`, `chats`, `notes`); на `feed` поиск центрируется не по шапке, а по колонке постов;
- справа — кнопка `← Назад` (на детальных экранах), `↑ К посту` (на `post`, появляется при прокрутке), переключатели режимов `Заметки` / `Чаты` (на `post`), меню `•••` с действиями.

Шапка реализована в стиле liquid glass: фиксирована поверх скролл-контейнера, контент уходит под неё с эффектом «жидкого стекла» (`backdrop-filter` + полупрозрачный фон, переменные `--glass-*` адаптируются под светлую и тёмную тему).

## Composer (поле ввода)

Composer одинаков по конструкции на `home`, `gchat` и `post` (внутри `chat`-режима). Отличия — только в наборе пунктов attach-меню.

Элементы строки управления (нижняя строка `input-bottom`):

- слева:
  - круглая кнопка `+` (универсальное attach-меню);
  - пилюля выбора LLM-модели (иконка `🧠` + название + `▾`);
  - пилюля выбора Web Search модели (иконка `🔎` + название + `▾`);
  - переключатель `Мультиответ` (когда в профиле включено `includeInMulti` у нескольких моделей).
- справа: круглая кнопка `↑` (отправить).

Поведение поля ввода:

- авто-ресайз вверх по контенту: на `home` до 10 строк, в `gchat`/`post`/`feed` — до 16 строк, дальше включается внутренний скролл;
- attach-меню открывается вниз на `home` и вверх в `gchat`/`post`/`feed` (там, где composer прижат к низу);
- attach-меню `Прикрепить пост` / `Медиа из прикреплённых постов` рендерятся через portal, имеют ограничение по высоте и собственный скролл.

Пины (`📌`) в composer’е больше нет — вся логика вложений собрана внутри `+`.

### Attach-меню `+`

Контекстные наборы пунктов:

- `home` — `Прикрепить пост` (список постов из ленты), `Прикрепить файл с компьютера`, `Медиа из прикреплённых постов` (показывается, когда к сообщению уже привязан хотя бы один пост с медиа);
- `gchat` — то же, что и на `home`;
- `post` — `Прикрепить пост`, `Медиа из поста` (медиа текущего поста), `Прикрепить файл с компьютера`, плюс `Медиа из прикреплённых постов`, когда привязаны другие посты;
- `feed` (создание черновика) — клик по `+` сразу открывает выбор файла с компьютера.

Медиа в подменю отображаются миниатюрами 72×72: 1–3 элемента в ряд по ширине контента, 4 элемента — сеткой 2×2, 5+ — три колонки со скроллом.

Прикреплённые посты и файлы показываются над textarea как «чипы»: их можно удалить крестиком или Backspace’ом при пустом поле. Также поддерживается синтаксис `@<title>;` — он автоматически превращает упоминание в чип.

Плейсхолдер по умолчанию: `Сообщение... введите @ чтобы прикрепить пост`.

## Сообщения в чатах (`home`, `gchat`, локальный `chat`)

- ответы ИИ: одиночный или мультиответ с переключателем вариантов; `AiMessageToolbar` — копирование, подсказка по моделям (LLM ± Web Search);
- пользовательские сообщения: редактирование по hover; при изменении текста создаётся **ветка** (`userBranches`), стрелки переключения между версиями;
- после правки пользовательского сообщения хвост ветки заменяется ответом-заглушкой (`STUB_REPLY_AFTER_USER_EDIT` в `lib/chatPaths.ts`).

## 1. `home` (новый глобальный чат)

**Назначение:** старт создания нового глобального чата.

**Элементы:**

- заголовок `Чем помочь сегодня?`;
- composer (см. раздел выше).

**Поведение:**

- отправка сообщения создаёт новый глобальный чат и открывает экран `gchat`;
- attach-меню открывается вниз.

---

## 2. `gchat` (открытый глобальный чат)

**Назначение:** переписка с глобальным ИИ.

**Шапка:**

- крошки `Чаты / Название чата` (слева);
- справа: `← Назад`, меню `•••` (`Удалить чат`).

**Тело:**

- история сообщений (поддерживается режим мультиответа с переключателем вариантов внутри карточки ИИ);
- composer внизу (attach-меню открывается вверх).

**Замечание:** визуального блока со списком закреплённых постов внутри чата нет — пины как сущность вынесены из этого экрана.

---

## 3. `feed` (лента)

**Назначение:** просмотр и создание постов.

**Шапка:**

- заголовок `Лента` (слева);
- поиск по постам в центре, выровнен по колонке карточек;
- справа — пусто (нет back-кнопки на верхнеуровневом экране).

**Структура:**

- секции `Опубликованные`, `Отложенные`, `Черновики` (заголовки центрированы над постами);
- карточка поста: опциональный TG-стиль медиа-блок сверху, текст, статус, дата, рубрика; у опубликованных — пилюли реакций, просмотры `👁`, репосты `↗`, счётчик комментариев `💬` (клик открывает комментарии поста);
- нижнее поле ввода для нового черновика (composer в режиме `feed`: `+` сразу открывает file picker, поддерживается несколько медиа в одном черновике).

**Действия:**

- `↑` внизу создаёт черновик;
- клик на карточку открывает `post`;
- drag-handle `⠿` у черновика позволяет перетащить карточку и изменить порядок в секции `Черновики`.

**Замечание:** порядок черновиков, изменённый drag-and-drop, хранится только в памяти вкладки (см. [12-demo.md](./12-demo.md)).

## 4. `post` (пространство поста)

**Назначение:** локальные чаты, локальные заметки и комментарии одного поста.

**Шапка:**

- крошки слева:
  - `Лента / Название` — в режиме `chat` без активного чата или в подрежимах;
  - `Лента / Название / Название чата` — в режиме `chat` с открытым локальным чатом;
  - `Лента / Название / Заметки` | `Чаты` | `Комментарии` — в соответствующих подрежимах;
- справа в порядке слева направо: `↑ К посту` (при прокрутке в `chat`), кластер `Заметки` (+ `+ Новая заметка` при активном режиме), кластер `Чаты` (+ `+ Новый чат` при активном режиме), `← Назад`, меню `•••`;
- `Заметки` и `Чаты` — двусостояние: активный режим подсвечен; повторный клик возвращает в `chat`;
- в режимах `chats`, `notes`, `comments` — поиск по центру шапки;
- `← Назад` работает по стеку переходов: переключение режимов и открытие чата кладёт состояние в стек, `← Назад` сначала разматывает стек и только при пустом стеке уходит на ленту.

**Меню `•••`:**

- `📝 Новый чат` — создаёт пустой черновик нового локального чата и переключает в режим `chat`;
- `📢 Опубликовать` (для черновика/отложенного);
- `🕐 Запланировать` (для черновика);
- `✕ Отменить публикацию` (для отложенного);
- `🗑 Удалить пост`.

**Режим `chat` (по умолчанию):**

- первое сообщение в скролле — карточка поста: TG-стиль медиа-блок над текстом + сам текст; у опубликованных — реакции, просмотры, репосты и строка «Комментарии» (переход в режим `comments`);
- inline-редактирование (`✏ Редактировать` → textarea авто-ресайзится от исходного размера, медиа остаётся сверху, есть кнопка `🖼 Добавить медиа`);
- ниже — история активного локального чата; при входе в пространство поста активного чата нет (composer пуст);
- первое отправленное сообщение создаёт локальный чат (заголовок берётся из первых ~40 символов), AI-ответ приходит в этот же чат;
- composer внизу (см. соответствующий раздел; `+` показывает посты, медиа из поста, файл с компьютера и медиа из других прикреплённых постов).

**Режим `chats`:**

- список карточек локальных чатов поста (название = первое сообщение, превью, дата);
- кнопка `+ Новый чат` сверху списка;
- `🗑` удаляет чат;
- клик по карточке открывает соответствующий чат в режиме `chat`.

**Режим `notes`:**

- сетка локальных заметок;
- поиск по заголовку и тексту;
- создание (`+ Новая заметка` в шапке), удаление, AI-флаг.

**Режим `comments`** (только у опубликованных постов):

- карточка поста (readonly) + список комментариев с поиском;
- ответ на комментарий (цитата автора), добавление нового комментария через `CommentComposer` внизу;
- вход также из ленты (клик по `💬` в метриках) или со строки «Комментарии» на карточке поста.

## 5. `note` (страница заметки)

**Назначение:** просмотр/редактирование глобальной или локальной заметки.

**Шапка:**

- крошки `Заметки / Название` (слева);
- справа: `← Назад`, меню `•••` (`Удалить заметку`);
- заголовок заметки авто-подгоняется под одну строку (`useFitTitleSize`).

**Общее:**

- два режима: `view` и `edit`;
- кнопка `Сохранить` есть в обоих режимах;
- при несохранённых изменениях показывается подтверждение при уходе.

**Глобальная заметка:**

- доступен переключатель `● Учитывать ИИ / ○ Учитывать ИИ`.

**Локальная заметка:**

- отображается метка `📌 Локальная` и ссылка `→ пост`.

**Вложения:**

- блок файлов (`NoteFilesPanel`): добавление с компьютера, встраивание в текст через `[имя файла]`, drag-and-drop в тело заметки.

---

## 6. `chats` (список чатов)

**Назначение:** доступ ко всем перепискам — глобальным и локальным.

**Шапка:**

- заголовок `Чаты`;
- поиск по чатам в центре.

**Секции:**

- вкладки `Глобальные` и `Локальные`.

**Действия:**

- клик по глобальному чату -> `gchat`;
- клик по локальному чату -> `post` с открытым именно этим локальным чатом (`currentPostChatId` устанавливается напрямую, без сброса);
- `→ пост` у локального чата;
- `🗑` удаляет чат (для глобальных — глобальный чат, для локальных — конкретный локальный чат поста).

**Замечание:** во вкладке `Локальные` каждая карточка соответствует **одному локальному чату**, а не посту. Один пост может иметь несколько локальных чатов и занимать несколько карточек; для контекста на карточке отображается название связанного поста.

---

## 7. `notes` (список заметок)

**Назначение:** единый каталог заметок.

**Шапка:**

- заголовок `Заметки`;
- поиск по заголовку и тексту в центре.

**Под шапкой:**

- scope-переключатель `Глобальные / Локальные`;
- фильтр `Все / В контексте ИИ / Не в контексте`;
- кнопка `+ Новая глобальная` в правом крае строки фильтров (видна только в scope `Глобальные`).

**Сетка:**

- карточки заметок.

**Действия:**

- открыть заметку;
- переключить AI-флаг;
- удалить заметку;
- перейти к посту из локальной заметки.

---

## 8. `analytics` (аналитика канала)

**Назначение:** краткая статистика канала.

**Элементы:**

- вкладки периода `7/30/90/Всё время`;
- карточки `Охваты`, `Подписчики`, `ER`;
- блок `По рубрикам` (bar-список);
- таблица `Лучшие посты за период`.

**Действия:**

- выбор периода (визуальное переключение таба);
- переход к посту из таблицы.

---

## 9. `profile` (профиль канала)

Три вкладки.

### `Канал`

- Целевая аудитория;
- Рубрики;
- Тон и голос;
- Запрещённые темы.

Подробная модель профиля — в [05-channel-profile.md](./05-channel-profile.md); в демо вкладка `Канал` показана упрощённо, без редактора.

### `Настройки`

- **Тема оформления** — сегментированный переключатель `☀ Светлая` / `🖥 Системная` / `🌙 Тёмная`; выбор сохраняется в `localStorage`, применяется через атрибут `data-theme` на `<html>`.
- **ИИ-движок** — список LLM-моделей (любую можно сделать активной, у каждой — провайдер через `ModelPicker`, поле API-ключа с явной рамкой, флаг `Включить в мультиответ`); аналогичный список Web Search моделей; общий переключатель `Мультиответ`; системный промпт в textarea с явной рамкой.
- **Telegram** — MTProto-форма: `api_id`, `api_hash`, телефон, код подтверждения, `sessionName`, выбор `syncMode` через `ModelPicker`, подключение/отключение канала.

Параметры `температура` и `max tokens` в текущем демо не вынесены в UI.

### `Аналитика платформы`

- Использование ИИ: запросы, токены, стоимость;
- Активность: чаты, заметки, посты.
