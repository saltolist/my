# Agent Actions Layer

## Идея кейса

`Agent Actions Layer` — это слой инструментов и интеграций для TG-платформы, который позволяет AI-ассистенту выполнять действия с контентом, а не ограничиваться текстовыми ответами. Агент анализирует запрос, выбирает нужный инструмент и завершает операцию в рамках сценария.

Кейс показывает переход от "assistant that answers" к "assistant that executes".

## Какую проблему решает

В типичной интеграции пользователь получает ответ, но действие все равно выполняется вручную. Это ломает автоматизацию и снижает ценность AI. `Agent Actions Layer` закрывает разрыв между решением и исполнением.

## Ключевые возможности

- Отдельный классификатор входящих сообщений, который разделяет обычные пользовательские сообщения и агентские команды.
- Каталог инструментов для работы с постами (редактирование, создание черновика, отложенная публикация, немедленная публикация, удаление).
- Планирование шагов выполнения для многокомандных задач.
- Валидация входных параметров перед вызовом инструмента.
- Политики безопасности и ограничения на действия.
- Обработка ошибок и откат/повтор шагов.
- Лог исполнения с понятной трассировкой "запрос -> шаги -> результат".

## Архитектурный контур

1. На входе классификатор определяет тип сообщения: обычный диалог или агентская команда.
2. Для обычных сообщений запрос остается в диалоговом контуре, без запуска action-пайплайна.
3. Для агентской команды планировщик определяет последовательность действий.
4. Action Layer вызывает нужные инструменты и интеграции.
5. Состояние задачи обновляется по каждому шагу.
6. Пользователь получает финальный результат и отчет о выполнении.

## Что показывается в демо

- Сценарий "отредактируй пост": агент вносит правки в текст по запросу пользователя.
- Сценарий "обычное сообщение vs агентская команда": классификатор корректно отделяет диалог от командного режима.
- Сценарий "запланируй публикацию": агент создает отложенный пост с датой и временем.
- Сценарий "опубликуй сейчас": агент публикует пост только после явного подтверждения пользователя.
- Сценарий "создай черновик": агент сохраняет пост в черновики для дальнейшей доработки.
- Сценарий "удали пост": удаление опубликованного, отложенного или чернового поста только после подтверждения пользователя.
- Ошибка на одном из шагов и корректная обработка без потери контекста.
- Прозрачный журнал действий агента в интерфейсе.

## Технологический фокус

- Tool-enabled agents
- Message/command classification
- Workflow orchestration
- Action safety and validation
- Execution tracing
- Resilient automation

## Практическая ценность

Инструмент позволяет автоматизировать контентные операции в TG-платформе через AI-ассистента, снижая количество ручных действий и добавляя безопасные подтверждения для критичных команд.

## Интеграция с AI Orchestrator Layer

Компонент может работать автономно со своей внутренней обработкой запроса: классификатор "обычное сообщение vs агентская команда", локальная маршрутизация и запуск действий.

Если в системе используется внешний обработчик запросов (например, `AI Orchestrator Layer`), входная логика компонента может быть делегирована ему, а `Agent Actions Layer` работает как специализированный исполнительный слой действий.

## Интеграция с Data Depersonalization Layer

Для корректной работы с пользовательскими данными рекомендуется подключать `Data Depersonalization Layer` в action-пайплайне до выхода в LLM и внешние API.

В production-сценариях предпочтительна связка `Data Depersonalization Layer` + `AI Orchestrator Layer`: это упрощает централизованную настройку безопасности и маршрутизации по сравнению с использованием только встроенных (родных) классификаторов и локальной оркестрации модуля.
