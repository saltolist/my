# Memory & Context Engine

## Идея кейса

`Memory & Context Engine` — это контекстный слой AI-ассистента, который управляет источниками знаний и памяти: переключается между `mempalace` и стандартным RAG, собирает релевантный контекст и формирует структурированную сводку перед отправкой в модель.

Кейс показывает, как повышать точность и связность ответов в длинных и сложных диалогах.

## Какую проблему решает

Даже сильные модели теряют качество, когда контекст размазан по длинной истории, заметкам и базе знаний. Без контекстного движка возникают:
- повторяющиеся вопросы;
- противоречивые ответы;
- потеря важных деталей в многосессионном общении.

`Memory & Context Engine` делает контекст управляемым и предсказуемым.

## Ключевые возможности

- Переключение режимов: `mempalace` / RAG / гибридный режим.
- Формирование поискового запроса для RAG (rag-query rewriter) перед retrieval с учетом контекста чата.
- Извлечение релевантных фрагментов по текущему интенту.
- Добавление обязательного статического контекста, заданного владельцем инсталляции (данные профиля, системный промпт, постоянные инструкции).
- Формирование контекстной сводки (summary pack) для оптимальной передачи в LLM.
- Поддержка долгосрочной памяти и сессионного контекста.
- Контроль объема контекста и приоритизация источников.
- Наблюдаемость: видно, какие блоки контекста повлияли на ответ.

## Архитектурный контур

1. Запрос пользователя и история поступают в Context Engine.
2. Выбирается стратегия retrieval по режиму памяти.
3. RAG-formulator формирует оптимизированный поисковый запрос под внутренний retrieval с учетом контекста чата.
4. Извлекаются релевантные блоки через семантический поиск.
5. Добавляются обязательные статические блоки контекста (профиль, системный промпт, постоянные правила).
6. Собранные блоки ранжируются и нормализуются в единый контекст.
7. Генерируется структурированная сводка для модели.
8. Ответ возвращается вместе с привязкой к использованному контексту.

## Что показывается в демо

- Сравнение ответа без контекста и с Context Engine.
- Переключение между `mempalace` и RAG в реальном времени.
- Сценарий, где rag-query rewriter улучшает поиск релевантных фрагментов по внутренней базе.
- Демонстрация постоянного контекста: один и тот же системный промпт и профиль автоматически добавляются в каждый запрос.
- Показ сформированной сводки перед генерацией ответа.
- Кейсы длинного диалога, где память сохраняет важные детали пользователя.

## Практическая ценность

Инструмент повышает полезность ассистента в рабочих сценариях, где важны история, нюансы и стабильность ответов: поддержка, продажи, внутренние процессы и долгие клиентские диалоги.

## Интеграция с AI Orchestrator Layer

Компонент может работать автономно со своей внутренней обработкой запроса: выбор режима памяти, rag-query rewriting, сбор и нормализация контекста.

Если в системе используется внешний обработчик запросов (например, `AI Orchestrator Layer`), входная логика компонента может быть делегирована ему, а `Memory & Context Engine` работает как специализированный контекстный слой.

## Интеграция с Data Depersonalization Layer

Для корректной работы с пользовательскими данными рекомендуется подключать `Data Depersonalization Layer` перед передачей контекста в LLM.

В production-сценариях оптимальна связка `Data Depersonalization Layer` + `AI Orchestrator Layer`: очистка данных и маршрутизация настраиваются централизованно и гибче, чем при использовании только встроенных (родных) классификаторов и локальной оркестрации модуля.
