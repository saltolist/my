# Omnichannel Sync (Web <-> Telegram)

## Идея кейса

`Omnichannel Sync` — это модуль синхронизации, где в веб-версии ассистента есть несколько чатов, но один из них постоянный и специально помечен как общий чат Web <-> Telegram. Именно этот чат синхронизируется с Telegram-ботом в обе стороны.

Пользователь может общаться в обычных веб-чатах отдельно, а "общий чат" использовать как единый канал между веб-платформой и Telegram без потери контекста и состояния.

Кейс демонстрирует омниканальную архитектуру AI-ассистента, а не просто отдельный бот или отдельный веб-чат.

## Какую проблему решает

Когда каналы не синхронизированы, бизнес сталкивается с:
- дублированием диалогов и ответов;
- потерей контекста при переходе между каналами;
- сложностью поддержки единого клиентского пути.

`Omnichannel Sync` устраняет разрыв между вебом и Telegram.

## Ключевые возможности

- Единый контекст диалога для Web и Telegram.
- Постоянный общий чат в веб-версии с отдельной пометкой канала синхронизации.
- Двусторонняя синхронизация сообщений в реальном времени.
- Маркировка источника сообщений (канал, сессия, направление).
- Обработка конфликтов при параллельной активности пользователя.
- Поддержка одного профиля пользователя в нескольких каналах.
- Надежная доставка и повторная отправка при сетевых сбоях.

## Архитектурный контур

1. Сообщение приходит из Web или Telegram.
2. Sync Layer нормализует событие в общий формат.
3. Если сообщение относится к "общему чату", оно обновляется в едином хранилище контекста.
4. Изменения транслируются между веб-версией и Telegram.
5. UI общего чата и бот отображают одинаковую актуальную историю.

## Что показывается в демо

- Старт диалога в вебе и продолжение в Telegram.
- Наличие нескольких чатов в вебе, где только один помечен как синхронизируемый с Telegram.
- Ответ из Telegram, который мгновенно появляется в веб-версии.
- Обратный сценарий: ответ из веба отображается в чате TG-бота.
- Прозрачная маркировка, откуда пришло каждое сообщение.

## Технологический фокус

- Event-driven synchronization
- Cross-channel conversation state
- Identity mapping
- Conflict handling
- Real-time updates

## Практическая ценность

Модуль можно внедрять в продукты, где важно общаться с пользователем в привычном ему канале, но сохранять единый опыт обслуживания и целостный AI-контекст.

## Интеграция с AI Orchestrator Layer

Компонент может работать автономно со своей внутренней обработкой транспортного контура: синхронизация событий, консистентность диалога и доставка сообщений между Web и Telegram.

Если в системе используется внешний обработчик запросов (например, `AI Orchestrator Layer`), входная логика обработки сообщений может быть делегирована ему, а `Omnichannel Sync` работает как специализированный слой кросс-канальной доставки и актуализации контекста.
