# Пространственная система (кратко)

Моя пространственная система - это архитектурная модель AI-платформы, где работа строится не вокруг одного универсального чата, а вокруг двух уровней контекста: глобального и локального. Ключевой принцип: ИИ работает в заданной зоне ответственности, а не "в общем режиме".

**Глобальное пространство** одно на проект. Это общий контур знаний и общий ИИ-помощник уровня всей системы. Здесь хранится стратегический слой: общие правила, профиль проекта, кросс-сущностные знания, глобальные заметки, агрегированная аналитика и другие артефакты, которые нужны для решений на уровне продукта целиком.

**Локальные пространства** привязаны к конкретной сущности: посту, документу, кейсу, сделке и т.д. Пользователь попадает в локальное пространство, когда открывает эту сущность. Локальный ИИ работает в фокусе ее данных, истории и задач, но использует глобальный контекст как фон.

Модель доступа асимметрична: глобальное пространство знает все, включая данные локальных пространств; локальное знает свое плюс глобальное. В текущей реализации локальные пространства не читают друг друга напрямую. В будущем допускается расширение через управляемый RAG-доступ между локальными пространствами.

Состав базы знаний пространства зависит от проекта: это могут быть заметки, документы, чаты, сущностные данные, правила, метрики и любые другие полезные артефакты. Конфигурация также гибкая: часть параметров задается разработчиком, часть может задаваться пользователем.

В каждом пространстве обычно есть основной ИИ-помощник, но при необходимости может включаться мультиагентная структура. Для критичных операций действует обязательное правило: действия агента выполняются по явной команде пользователя.

Эта система дает три эффекта: управляемый контекст, предсказуемое поведение ИИ и масштабируемость без архитектурного хаоса.
