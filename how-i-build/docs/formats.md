# Форматы сотрудничества

Фиксированная цена, Time and Material, ретейнер.

---

## Как выбирается формат

Формат сотрудничества выбирается не "по привычке", а по уровню неопределенности проекта, скорости изменений и типу результата.

Основной критерий: формат должен помогать принимать решения по данным, а не создавать конфликт ожиданий.

## Fixed Price (фиксированная цена)

Подходит, если:

- scope четко определен;
- изменения минимальны;
- критерии приемки однозначны;
- риски интеграций заранее понятны.

Плюсы:

- прозрачный бюджет;
- понятные границы результата;
- проще планировать запуск.

Ограничения:

- хуже переносит частые изменения;
- требует качественного Discovery до старта;
- может тормозить гибкость в процессе.

## Time and Material (T&M)

Подходит, если:

- проект итеративный;
- есть высокая неопределенность;
- приоритеты могут меняться по мере проверки гипотез.

Плюсы:

- максимальная гибкость;
- проще адаптироваться к новым данным;
- удобен для AI-продуктов с короткими циклами итераций.

Ограничения:

- нужен прозрачный контроль прогресса и бюджета;
- важно регулярно пересматривать приоритеты.

## Retainer (поддержка/развитие по подписке)

Подходит, если:

- продукт уже живет в проде;
- нужна регулярная доработка и поддержка;
- важно быстро реагировать на новые задачи и инциденты.

Плюсы:

- стабильный ритм улучшений;
- предсказуемый доступ к ресурсу;
- меньше накладных расходов на постоянный "перезапуск" работ.

Ограничения:

- нужен общий долгосрочный фокус;
- важно заранее определить объем и SLA реакции.

## Гибридные варианты

На практике часто работает комбинация:

- фикс для четко очерченного стартового объема;
- затем T&M для итераций;
- после релиза ретейнер для стабильной поддержки и роста.

Это позволяет совместить предсказуемость старта и гибкость развития.

## Что фиксируем независимо от формата

В любом формате заранее согласуем:

- цели и критерии успеха;
- зоны ответственности;
- правила коммуникации и отчетности;
- порядок работы с изменениями;
- условия завершения этапа и приемки.

Без этих правил формат сам по себе не спасает проект.

## Как обычно выбираем формат на старте

Простой ориентир:

- четкий scope + низкая неопределенность -> Fixed;
- evolving scope + быстрые гипотезы -> T&M;
- действующий продукт + постоянные задачи -> Retainer.

Если проект смешанный, выбираем гибрид по этапам.

## Анти-паттерны

- выбирать fixed price при плавающем scope;
- идти в T&M без прозрачного трекинга;
- ожидать 24/7 сопровождения из базового ретейнера;
- не фиксировать условия изменений и приемки;
- менять формат в середине без явного пересогласования.

## Практический checklist

Перед стартом формата проверяю:

- понятен уровень неопределенности проекта;
- согласованы бюджетные ограничения;
- определены правила отчетности и коммуникации;
- есть механизм управления изменениями;
- зафиксированы критерии результата для этапа.
